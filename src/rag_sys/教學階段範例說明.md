# RAG AI 引導教學系統 - 完整教學範例

## 📚 教學流程總覽

本系統採用**階段式引導教學**，從核心概念開始，逐步引導學生理解，最終達到完全掌握（99分）。

**5個學習階段：**
1. **核心概念確認** (0-39分) - 評估對題目核心概念的掌握
2. **相關概念引導** (40-69分) - 圍繞核心概念擴展相關知識
3. **應用理解** (70-89分) - 將理解應用到實際情境
4. **理解驗證** (90-98分) - 學生用自己的話解釋，AI修正錯誤
5. **學習完成** (99分) - 完全掌握，可進入下一題

---

## 🎯 完整教學範例：資料庫正規化

### 📝 題目
**題目：** 什麼是關聯式資料庫的正規化？請說明第一正規化（1NF）的目的。

**學生初始答案：** 正規化就是把資料整理好，1NF 就是不要有重複的資料。

---

### 階段 1：核心概念確認 (0-39分)

**🎯 AI 提問：**
```
同學，我看到你對正規化的概念還不太清楚。讓我們從最核心的概念開始：

**問題 1：** 在關聯式資料庫中，如果一個資料表中有重複的資料列，會產生什麼問題？請舉一個具體的例子說明。

評分：25分
```

**👤 學生回答：**
```
重複的資料列會造成資料冗餘，浪費儲存空間。例如，如果一個訂單表中有多個商品，每個商品都重複記錄客戶資訊，這樣會浪費空間。
```

**🎯 AI 回應：**
```
很好！你理解了資料冗餘的問題。但除了空間浪費，還有其他問題嗎？比如，如果客戶的地址改變了，你需要修改多少筆記錄？這會造成什麼問題？

評分：35分
```

**👤 學生回答：**
```
如果客戶地址改變，需要修改所有相關的訂單記錄。如果漏掉某些記錄，就會造成資料不一致的問題。
```

**🎯 AI 回應：**
```
非常好！你現在理解了資料冗餘會導致：
1. 儲存空間浪費
2. 資料更新困難
3. 資料不一致的風險

這就是為什麼需要正規化。現在讓我們看看第一正規化（1NF）如何解決這些問題。

評分：38分
```

---

### 階段 2：相關概念引導 (40-69分)

**🎯 AI 提問：**
```
很好！你已經理解了資料冗餘的問題。現在讓我們深入第一正規化（1NF）的核心概念：

**問題 2：** 第一正規化（1NF）要求每個欄位都必須是「原子值」（atomic value）。什麼是原子值？請舉一個違反 1NF 的例子，並說明如何修正。

評分：55分
```

**👤 學生回答：**
```
原子值就是不能再分割的值。例如，如果一個「地址」欄位包含「台北市信義區信義路五段7號」，這就不是原子值，因為它包含了城市、區、路名和號碼。應該拆分成「城市」、「區」、「路名」、「號碼」等欄位。
```

**🎯 AI 回應：**
```
非常正確！你完全理解了原子值的概念。但 1NF 還有一個重要要求：**不能有重複的資料列**。讓我換個方式問：

如果一個「學生選課」表包含：學號、姓名、課程代碼、課程名稱、學分
當一個學生選了多門課時，會出現什麼問題？這符合 1NF 嗎？

評分：62分
```

**👤 學生回答：**
```
如果一個學生選了多門課，姓名會重複出現。這不符合 1NF，因為有重複的資料列。應該拆分成「學生」表和「選課」表，用學號作為外鍵關聯。
```

**🎯 AI 回應：**
```
完美！你現在完全理解了 1NF 的兩個核心要求：
1. 每個欄位必須是原子值
2. 不能有重複的資料列

這就是 1NF 如何解決資料冗餘和資料不一致的問題。

評分：68分
```

---

### 階段 3：應用理解 (70-89分)

**🎯 AI 提問：**
```
很好！你已經掌握了 1NF 的核心概念。現在讓我們應用到實際情境：

**問題 3：** 假設你有一個「學生選課」資料表，包含以下欄位：
- 學號、姓名、課程代碼、課程名稱、學分、成績

這個資料表是否符合 1NF？如果不符合，請說明如何修正，並解釋修正後如何解決資料冗餘和資料不一致的問題。

評分：82分
```

**👤 學生回答：**
```
這個資料表不符合 1NF，因為：
1. 如果一個學生選了多門課，姓名會重複出現（重複資料列）
2. 如果課程名稱改變，需要修改多筆記錄（資料不一致）

修正方式：
- 拆分成「學生」表：學號、姓名
- 「選課」表：學號、課程代碼、成績
- 「課程」表：課程代碼、課程名稱、學分

這樣可以避免姓名重複，課程資訊改變時只需要修改課程表，不會影響選課記錄。
```

**🎯 AI 回應：**
```
非常正確！你不僅理解了 1NF 的概念，還能正確應用到實際情境。你的修正方案完全符合 1NF 的要求，並且清楚地說明了如何解決資料冗餘和資料不一致的問題。

評分：88分
```

---

### 階段 4：理解驗證（反向教導）(90-98分 → 99分)

**🎯 AI 提問：**
```
很好！你已經達到了 90 分。現在進入理解驗證階段：

**問題 4：** 請你用自己的話，完整地向我解釋：
1. 什麼是第一正規化（1NF）？
2. 為什麼需要 1NF？
3. 1NF 如何解決資料冗餘和資料不一致的問題？
4. 請舉一個實際例子說明。

請確保你的解釋清晰、完整，並且用你自己的理解來表達。

評分：92分
```

**👤 學生回答：**
```
1NF 就是要求每個欄位都必須是單一值，不能分割，而且不能有重複的資料列。需要 1NF 是因為如果資料表設計不好，會有資料冗餘和資料不一致的問題。1NF 透過拆分資料表來解決這些問題。例如，學生選課表如果包含學生資訊和課程資訊，就會有重複，應該拆分成學生表、選課表和課程表。
```

**🎯 AI 回應：**
```
很好！你理解了 1NF 的核心概念。但有一個小地方需要更精確：

你說「不能有重複的資料列」，這個說法不夠準確。1NF 要解決的是「**功能相依性**」的問題。讓我再問一次：

如果一個資料表有「學號、姓名、課程代碼、課程名稱」，為什麼「姓名」會重複？是因為「姓名」依賴於什麼？而「課程名稱」又依賴於什麼？

評分：95分
```

**👤 學生回答：**
```
姓名會重複是因為「姓名」依賴於「學號」，而不是依賴於「課程代碼」。同樣地，「課程名稱」依賴於「課程代碼」，而不是依賴於「學號」。所以如果把它們放在同一個表裡，就會造成重複和資料不一致的問題。
```

**🎯 AI 回應：**
```
完美！你現在完全理解了功能相依性的概念。這就是 1NF 的核心：**每個非主鍵欄位都必須完全依賴於主鍵**。

如果「姓名」只依賴於「學號」，而「課程名稱」只依賴於「課程代碼」，那麼把它們放在同一個表裡就會違反 1NF。

你的理解非常深刻！

評分：99分
```

---

### 階段 5：學習完成 (99分)

**🎯 AI 回應：**
```
🎉 恭喜！你已經完全理解了第一正規化（1NF）的核心概念、功能相依性的原理，以及如何應用到實際情境中。

你的理解程度達到了 99 分，可以進入下一題了。

如果你還有其他問題，隨時可以問我。讓我們繼續學習吧！

評分：99分
```

## 📊 教學流程總結

### 階段進展軌跡
```
階段 1：核心概念確認
  25分 → 35分 → 38分
  ↓
階段 2：相關概念引導
  55分 → 62分 → 68分
  ↓
階段 3：應用理解
  82分 → 88分
  ↓
階段 4：理解驗證（反向教導）
  92分 → 95分 → 99分
  ↓
階段 5：學習完成
  99分 ✅
```

### 🔑 關鍵教學原則

1. **從核心開始**：從「資料冗餘問題」這個與題目最直接相關的核心概念開始，而不是從「什麼是資料庫」這種基礎概念開始

2. **概念連貫性**：每個問題都圍繞「1NF 如何解決資料冗餘」這個核心，沒有跳脫到不相關的概念

3. **蘇格拉底式提問**：透過引導性問題（「除了空間浪費，還有其他問題嗎？」），讓學生自己思考並得出答案

4. **精確評分**：每次學生回答後，都給出具體的 0-100 分評分，評估理解程度

5. **理解驗證**：在 90 分後進入反向教導階段，要求學生用自己的話解釋，並持續修正錯誤直到 99 分

6. **直接給答案**：當學生理解不清楚時（如功能相依性），直接給出正確答案和解釋，而不是重複問「你知道嗎？」

## 📝 評分邏輯說明

- **25分**：理解資料冗餘的基本概念，但理解不深
- **35分**：理解資料冗餘會造成多個問題
- **38分**：完全理解資料冗餘的問題，可以進入下一階段
- **55分**：理解原子值的概念
- **62分**：理解 1NF 的兩個核心要求
- **68分**：完全掌握 1NF 的核心概念
- **82分**：能正確應用到實際情境
- **88分**：應用正確且解釋清晰
- **92分**：能用自己話解釋，但有小錯誤
- **95分**：理解更深入，接近完全掌握
- **99分**：完全理解，包括功能相依性的深層概念

## 💡 這個範例的特色

1. **完整展現五個階段**：從 25 分到 99 分的完整教學流程
2. **評分逐步提升**：每個階段都有明確的評分進展
3. **概念逐步深入**：從表面的「資料冗餘」到深層的「功能相依性」
4. **實際應用驗證**：在應用理解階段有具體的資料表設計案例
5. **反向教導展現**：在理解驗證階段展現 AI 如何修正學生的錯誤理解
6. **教學原則體現**：每個階段都體現了「從核心開始」、「概念連貫性」等教學原則
