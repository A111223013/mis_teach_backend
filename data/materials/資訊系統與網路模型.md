# 資訊系統與網路模型

本章節將深入探討資訊系統的核心概念，以及電腦網路運作的基石——網路模型。理解這兩者對於在數位時代有效運用與管理資訊至關重要。

-----

### 資訊系統 (Information Systems, IS)

#### 核心概念：資訊系統的定義與組成

**定義：** 資訊系統 (Information System, IS) 是一個由人、硬體、軟體、資料和網路（或稱為流程）所組成的整合系統，其主要目的是收集、處理、儲存、分析、分發資訊，以支援組織的營運、管理、分析及決策制定。資訊系統不僅僅是技術的堆疊，更是一種社會技術系統 (Socio-technical System)，它將技術與組織中的人、任務和結構緊密結合。

**組成要素 (五大組件)：**

1.  **硬體 (Hardware)：** 構成資訊系統的實體設備，包括電腦、伺服器、網路設備（路由器、交換器）、儲存設備、輸入/輸出設備（鍵盤、滑鼠、印表機）等。
2.  **軟體 (Software)：** 指揮硬體運作的程式和指令集合。可分為：
    *   **系統軟體：** 作業系統 (如 Windows, Linux)、資料庫管理系統 (DBMS)。
    *   **應用軟體：** 特定業務功能的程式，如文書處理、電子郵件、企業資源規劃 (ERP) 系統、客戶關係管理 (CRM) 系統等。
3.  **資料 (Data)：** 系統處理的原始事實、數字、文字、圖像、聲音等未經組織的原始材料。透過系統處理後，資料轉化為有意義的資訊。
4.  **人 (People)：** 資訊系統的設計者、開發者、管理者、維護者和最終使用者。人的參與是系統成功的關鍵。
5.  **流程 (Process/Network)：**
    *   **流程：** 指的是資訊系統如何處理資料、產生資訊的步驟與規則。它定義了資料的流向、轉換和使用方式。例如，訂單處理流程、庫存管理流程。
    *   **網路：** 指的是將硬體、軟體、資料和人連接起來的通訊基礎設施，使得資訊可以在不同的設備和使用者之間傳輸和共享。在某些定義中，網路被視為連結硬體和軟體以實現資料共享的關鍵要素。

**資訊系統的功能：**

*   **輸入 (Input)：** 收集原始資料。
*   **處理 (Processing)：** 將原始資料轉換為有意義的資訊。
*   **輸出 (Output)：** 將處理後的資訊呈現給使用者。
*   **儲存 (Storage)：** 儲存資料和資訊以供未來使用。
*   **回饋 (Feedback)：** 將系統輸出資訊重新輸入到系統中，以進行調整或修正。

#### 典型例子與轉換：資訊系統的實際應用

資訊系統無處不在，廣泛應用於各行各業：

*   **企業資源規劃 (ERP) 系統：** 整合企業內部所有功能部門（如財務、會計、生產、採購、銷售、人力資源）的資訊，實現資料共享和業務流程自動化。
    *   **例子：** SAP, Oracle ERP Cloud。
*   **客戶關係管理 (CRM) 系統：** 用於管理企業與客戶之間互動的系統，旨在改善客戶服務、促進銷售並建立更穩固的客戶關係。
    *   **例子：** Salesforce, Microsoft Dynamics 365。
*   **供應鏈管理 (SCM) 系統：** 管理從原材料採購到最終產品交付給客戶的整個供應鏈流程，以提高效率和降低成本。
    *   **例子：** SAP SCM, Oracle SCM Cloud。
*   **銷售點 (POS) 系統：** 零售業中用於處理交易、管理庫存和追蹤銷售的系統。
    *   **例子：** 各大超市和商店使用的收銀系統。
*   **圖書館管理系統：** 負責圖書的編目、借閱、歸還、讀者管理等功能。
*   **線上學習管理系統 (LMS)：** 提供課程內容、作業提交、成績管理、師生互動等功能的平台。
    *   **例子：** Moodle, Canvas, Google Classroom。

### 網路模型 (Network Models)

#### 核心概念：網路模型及其重要性

**定義：** 網路模型是一種抽象的架構，用於描述電腦網路中不同功能和協定如何組織、運作和相互通訊。它將複雜的網路通訊過程劃分為多個獨立的層次，每一層負責特定的功能，並與其上下層進行互動。這種分層設計有助於標準化、簡化複雜性、促進互通性，並使得網路技術的開發和維護更加高效。

#### 典型網路模型：OSI 模型與 TCP/IP 模型

目前最廣為人知的網路模型有兩種：OSI (Open Systems Interconnection) 模型和 TCP/IP (Transmission Control Protocol/Internet Protocol) 模型。

##### 1. OSI 模型 (開放系統互連模型)

OSI 模型是由國際標準化組織 (ISO) 提出的概念性模型，它將網路通訊功能劃分為七個層次。儘管它在實務上並未被廣泛實作，但它為理解網路通訊提供了一個標準化的框架。

| 層次 (Layer) | 名稱 (中文)        | 主要功能與協定舉例                                                                                                                                              |
| :----------- | :----------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 7            | **應用層** (Application Layer) | 提供網路服務給使用者應用程式，如電子郵件、檔案傳輸、網頁瀏覽。 <br/> *協定：HTTP, FTP, SMTP, DNS*                                                          |
| 6            | **表達層** (Presentation Layer) | 處理資料的編碼、加密、壓縮，確保不同系統間資料格式的一致性。 <br/> *協定：JPEG, MPEG, SSL/TLS (部分)*                                                     |
| 5            | **會話層** (Session Layer)       | 管理應用程式之間的通訊會話，包括建立、管理和終止連線。 <br/> *協定：NetBIOS, RPC*                                                                       |
| 4            | **傳輸層** (Transport Layer)     | 提供端到端 (End-to-End) 的資料傳輸服務，負責資料分段、重組、流量控制和錯誤恢復。 <br/> *協定：TCP (可靠傳輸), UDP (不可靠傳輸)*                              |
| 3            | **網路層** (Network Layer)       | 處理資料包的邏輯定址和路由選擇，將資料從來源主機傳送到目的主機。 <br/> *協定：IP (網際網路協定), ICMP (網際網路控制訊息協定)*                                |
| 2            | **資料連結層** (Data Link Layer) | 處理實體層上資料的傳輸，負責幀的同步、錯誤檢測與修正，以及媒體存取控制 (MAC)。 <br/> *協定：Ethernet (乙太網路), PPP (點對點協定), ARP (位址解析協定)* |
| 1            | **物理層** (Physical Layer)      | 定義傳輸介質、電氣特性、機械規範和程序，處理原始位元的傳輸。 <br/> *介質：光纖、銅線、無線電波*                                                         |

##### 2. TCP/IP 模型 (傳輸控制協定/網際網路協定模型)

TCP/IP 模型是網際網路的基礎，它在實務中被廣泛應用。它將OSI的七層簡化為四個主要層次。

| 層次 (Layer)             | 名稱 (中文)          | 主要功能與協定舉例                                                                                             | OSI 模型對應層次                                            |
| :----------------------- | :------------------- | :------------------------------------------------------------------------------------------------------------- | :---------------------------------------------------------- |
| 4                        | **應用層** (Application Layer) | 負責特定應用程式的數據處理，提供使用者介面與服務。 <br/> *協定：HTTP, FTP, SMTP, DNS, SSH, Telnet*       | 應用層 (7), 表達層 (6), 會話層 (5)                        |
| 3                        | **傳輸層** (Transport Layer)   | 負責端到端的資料傳輸，提供可靠或不可靠的連接服務。 <br/> *協定：TCP (可靠傳輸), UDP (不可靠傳輸)*           | 傳輸層 (4)                                                |
| 2                        | **網路層** (Internet Layer)    | 負責將數據包從來源傳送到目的地，處理IP定址和路由選擇。 <br/> *協定：IP (網際網路協定), ICMP, ARP*       | 網路層 (3)                                                |
| 1                        | **網路存取層** (Network Access Layer) | 負責數據在實體網路上的實際傳輸，包括物理介質的連接、數據幀的創建與傳輸。 <br/> *協定：Ethernet, Wi-Fi* | 資料連結層 (2), 物理層 (1) (有時也稱資料連結層和物理層) |

#### 與相鄰概念的關聯

*   **資訊系統與網路模型：** 網路模型是資訊系統實現其通訊功能的基石。現代的資訊系統大多是分散式的，需要透過網路進行資料交換與協同工作。網路模型定義了這些通訊如何進行，確保不同資訊系統組件（例如客戶端與伺服器、不同部門的系統）之間能夠順暢地傳遞資訊。
*   **網路模型與網路設備：** 路由器、交換器、集線器等網路設備都是根據網路模型的分層概念設計的。
    *   **集線器 (Hub) / 中繼器 (Repeater)：** 工作在**物理層**，負責訊號的放大與轉發。
    *   **交換器 (Switch) / 橋接器 (Bridge)：** 工作在**資料連結層**，根據 MAC 位址轉發數據幀。
    *   **路由器 (Router)：** 工作在**網路層**，根據 IP 位址轉發資料包，實現跨網路通訊。
    *   **防火牆 (Firewall) / 負載平衡器 (Load Balancer)：** 可以工作在**傳輸層**及更高層次，對通訊進行更細緻的控制和管理。

### 進階內容：資訊系統類型與網路拓撲

#### 資訊系統的常見類型

除了依功能劃分的 ERP、CRM 等，資訊系統也可依其支援的組織層級和決策性質進行分類：

1.  **交易處理系統 (Transaction Processing Systems, TPS)：**
    *   **目的：** 處理日常、重複性的業務交易，如訂單輸入、工資處理、銷售紀錄。
    *   **特點：** 處理大量數據、高效率、準確性。
    *   **層級：** 支援底層營運管理。
2.  **管理資訊系統 (Management Information Systems, MIS)：**
    *   **目的：** 將 TPS 產生的原始數據匯總、整理，生成標準化報告，協助中層管理者進行日常監督和控制。
    *   **特點：** 提供固定格式的報告，支援預先定義的決策。
    *   **層級：** 支援中層管理。
3.  **決策支援系統 (Decision Support Systems, DSS)：**
    *   **目的：** 提供互動式工具和分析模型，協助中高層管理者解決非結構化或半結構化的決策問題。
    *   **特點：** 靈活性高，能進行「假設分析」(What-if analysis)。
    *   **層級：** 支援中高層管理。
4.  **主管資訊系統 (Executive Information Systems, EIS) / 商業智慧 (Business Intelligence, BI)：**
    *   **目的：** 提供高度匯總的關鍵績效指標 (KPI) 和趨勢分析，協助高層主管進行策略規劃和長期決策。
    *   **特點：** 視覺化界面、易於使用、外部資訊整合。
    *   **層級：** 支援高層管理。

#### 網路拓撲 (Network Topology)

網路拓撲描述了網路中電腦和設備的物理或邏輯連接方式，它影響了網路的效能、可靠性和成本。

*   **匯流排拓撲 (Bus Topology)：** 所有設備共享一條中央電纜，訊號沿著電纜傳播。
    *   **優點：** 線材少，成本低。
    *   **缺點：** 一個故障可能導致整個網路中斷；故障排除困難；安全性較差。
*   **星狀拓撲 (Star Topology)：** 所有設備都連接到一個中央集線器或交換器。
    *   **優點：** 容易安裝和管理；一個設備故障不影響其他設備；故障排除容易。
    *   **缺點：** 中央設備故障會導致整個網路癱瘓；線材較多。
*   **環狀拓撲 (Ring Topology)：** 每個設備連接到其相鄰的兩個設備，形成一個環。
    *   **優點：** 每個設備有平等的存取權。
    *   **缺點：** 任何一個節點或纜線故障都可能導致整個網路中斷；擴展性差。
*   **網狀拓撲 (Mesh Topology)：** 每個設備都直接連接到網路中的其他所有設備。
    *   **優點：** 高度冗餘和可靠性；一個連接故障不影響通訊。
    *   **缺點：** 佈線複雜、成本高昂；通常用於核心網路或高可用性需求環境。

### 常見錯誤與澄清

1.  **誤解 OSI 與 TCP/IP 是替代品：**
    *   **澄清：** OSI 是一個**概念性、理論性**的模型，旨在提供一個標準化的通訊框架，它定義了網路通訊的七個邏輯層次。TCP/IP 是一個**實務性、操作性**的模型和協定套件，是網際網路的基礎。兩者不是互相取代的關係，而是 TCP/IP 在設計和實作上參考了 OSI 的分層思想，並在實際應用中證明其高效性。OSI 更像一個藍圖，TCP/IP 是基於這個藍圖或類似思想建造的實際建築。
2.  **將網路模型中的「層」與硬體或軟體功能混淆：**
    *   **澄清：** 網路模型中的每一層代表的是**邏輯職責的劃分**，而不是指特定的硬體設備或軟體程式。一個網路設備可能跨越多個層次的功能（例如，多層交換器可以在資料連結層和網路層之間轉發數據），一個軟體應用程式也可能使用多個層次的協定。層次劃分是為了管理和理解複雜的通訊過程。
3.  **認為資訊系統只是電腦和軟體：**
    *   **澄清：** 資訊系統不僅包含硬體和軟體，更重要的是**資料、人**和**流程**。一個沒有經過良好設計的流程，或沒有受過訓練的使用者，即使有最先進的硬體和軟體，也無法構成一個有效的資訊系統。資訊系統是一個**社會技術系統**，強調人與技術的整合。

### 小練習（附詳解）

#### 小練習一：資訊系統組件識別

**情境：** 某大學導入了一套「學生選課系統」，學生可以在線上瀏覽課程、提交選課申請、查詢選課結果。教務處則使用該系統管理課程開設、選課人數限制、核准選課等。

**問題：** 請列出該「學生選課系統」的五個核心組件，並為每個組件提供一個具體的例子。

**詳解：**

1.  **硬體 (Hardware)：**
    *   **具體例子：** 運行選課系統的**伺服器主機**、學生和教務處使用的**個人電腦或平板**、連接這些設備的**網路路由器和交換器**、儲存選課數據的**儲存陣列 (Storage Array)**。
2.  **軟體 (Software)：**
    *   **具體例子：**
        *   **系統軟體：** 伺服器上運行的 **Linux 作業系統**、用於管理學生和課程資料的 **MySQL 資料庫管理系統**。
        *   **應用軟體：** 開發選課系統的 **Java 程式碼**、提供學生和教務處介面的 **Web 應用程式**。
3.  **資料 (Data)：**
    *   **具體例子：** **學生基本資料**（學號、姓名、系所）、**課程資訊**（課程代碼、名稱、學分、上課時間地點、教師）、**選課紀錄**（學生選了哪些課、選課狀態）、**成績資料**。
4.  **人 (People)：**
    *   **具體例子：**
        *   **使用者：** **大學生**（進行選課）、**教務處人員**（管理課程與選課）。
        *   **維護者：** 負責系統正常運作的**資訊技術人員**、負責系統更新和功能開發的**軟體工程師**。
5.  **流程 (Process)：**
    *   **具體例子：**
        *   **選課流程：** 學生登入系統 -> 瀏覽課程 -> 提交選課 -> 系統檢查選課條件 -> 教務處核准 -> 系統更新選課結果。
        *   **課程開設流程：** 系所提交課程申請 -> 教務處審核 -> 系統錄入課程資訊。

#### 小練習二：網路協定與模型層次歸類

**情境：** 小明在瀏覽器中輸入 `https://www.google.com` 後，成功看到 Google 搜尋頁面。這個簡單的動作涉及到網路模型中多個層次的協同工作。

**問題：** 請從網路通訊的角度，說明這個過程中最少涉及的兩個 TCP/IP 模型層次，並為每個層次舉出一個相關的協定。

**詳解：**

1.  **應用層 (Application Layer)：**
    *   **說明：** 當小明輸入 `https://www.google.com` 時，瀏覽器作為應用程式需要與網頁伺服器進行通訊。應用層負責提供這種使用者可見的服務。
    *   **相關協定：**
        *   **HTTPS (Hypertext Transfer Protocol Secure)：** 這是用於安全地傳輸超文本（網頁內容）的協定。它在 HTTP 的基礎上增加了 SSL/TLS 加密，確保數據傳輸的安全性。
        *   **DNS (Domain Name System)：** 在瀏覽器發送 HTTPS 請求之前，需要將 `www.google.com` 這個域名解析為其對應的 IP 位址。DNS 協定負責這項域名解析服務。
2.  **傳輸層 (Transport Layer)：**
    *   **說明：** 在應用層確定要連接的伺服器和協定後，傳輸層負責建立端到端（小明的電腦到 Google 伺服器）的通訊連接。它將應用層數據分割成段 (segments)，並處理這些段的傳輸。
    *   **相關協定：**
        *   **TCP (Transmission Control Protocol)：** 由於網頁瀏覽需要可靠的數據傳輸（確保所有數據都按順序到達且沒有丟失），TCP 被選用來提供這種可靠的連接導向服務。它負責數據包的排序、錯誤檢測與重傳。
3.  **網路層 (Internet Layer)：**
    *   **說明：** 傳輸層將數據段傳遞給網路層。網路層的主要職責是處理邏輯位址 (IP 位址) 和路由，確保數據包能夠從來源主機（小明的電腦）透過多個網路路由器，最終抵達目的地主機（Google 的伺服器）。
    *   **相關協定：**
        *   **IP (Internet Protocol)：** 這是網際網路的核心協定，負責數據包的定址和路由。每個數據包都會帶有來源 IP 和目的 IP，路由器依此將數據包轉發到正確的路徑上。
4.  **網路存取層 (Network Access Layer)：**
    *   **說明：** 網路層的數據包最終會被傳遞給網路存取層，該層負責將數據轉換成可以在物理介質（如乙太網路線、Wi-Fi 電波）上傳輸的訊號。它處理物理位址 (MAC 位址) 和媒體存取控制。
    *   **相關協定：**
        *   **Ethernet (乙太網路) 或 Wi-Fi (無線區域網路)：** 根據小明的電腦是透過有線還是無線連接到路由器，會使用對應的物理層和資料連結層協定來傳輸數據。

**總結：** 僅僅瀏覽一個網頁，就至少涉及了 TCP/IP 模型中的應用層 (HTTPS/DNS)、傳輸層 (TCP)、網路層 (IP) 和網路存取層 (Ethernet/Wi-Fi) 的協同工作。
