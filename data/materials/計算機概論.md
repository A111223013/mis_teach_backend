# 計算機概論：核心概念與基本原理

本章將引導您進入計算機的奇妙世界。我們將從計算機的本質定義出發，逐步解析其內部運作的基石，包括資料的表示方式，以及硬體與軟體兩大核心組成。理解這些基礎知識，將為您後續學習更進階的計算機科學打下堅實的基礎。

### 1. 什麼是計算機？

#### 核心概念：定義與功能

計算機（Computer）是一種能夠依據指令對資料進行算術及邏輯運算的電子設備。它設計用於自動化處理資訊，並且具有輸入、處理、儲存和輸出資料的能力。

*   **輸入 (Input)**：接收來自外部世界的資料或指令。例如：鍵盤輸入文字、滑鼠點擊、麥克風錄音。
*   **處理 (Processing)**：對輸入的資料進行運算、分析、轉換。這是計算機的核心活動，由中央處理器（CPU）負責。
*   **儲存 (Storage)**：保留資料和程式以供將來使用。儲存分為暫時性（如記憶體）和永久性（如硬碟）。
*   **輸出 (Output)**：將處理後的結果呈現給使用者或傳送給其他設備。例如：螢幕顯示圖像、印表機列印文件、喇叭播放聲音。

-----

#### 典型例子與應用

計算機的形式多樣，從微型嵌入式系統到超級電腦，無處不在：

*   **個人電腦 (Personal Computer, PC)**：包括桌上型電腦和筆記型電腦，用於個人辦公、娛樂和學習。
*   **智慧型手機與平板電腦 (Smartphones & Tablets)**：行動運算的代表，整合了通訊、娛樂和生產力功能。
*   **伺服器 (Servers)**：通常運行在資料中心，為網路上的其他電腦（客戶端）提供服務，如網站託管、資料庫管理。
*   **嵌入式系統 (Embedded Systems)**：集成在其他設備中的專用計算機系統，如智慧家電、汽車電子、醫療儀器。

-----

#### 與相鄰概念的關聯

計算機是「資訊科技 (Information Technology, IT)」領域的核心支柱。它與「資料 (Data)」緊密相連，因為計算機的本質就是處理資料。而「資訊 (Information)」則是經過計算機處理、有意義的資料。沒有計算機，現代的資料處理、通訊和人工智慧等進階領域都將無法實現。

-----

### 2. 計算機的基本工作原理：馮紐曼架構

#### 核心概念：統一儲存與指令執行週期

現代計算機的設計基礎多數遵循由約翰·馮·紐曼（John von Neumann）於1945年提出的「馮紐曼架構 (Von Neumann Architecture)」。這個架構的核心思想是：**程式指令和資料都儲存在同一記憶體空間中**。

馮紐曼架構的主要組成部分包括：

*   **中央處理器 (Central Processing Unit, CPU)**：計算機的「大腦」，負責執行指令和進行運算。它包含：
    *   **算術邏輯單元 (Arithmetic Logic Unit, ALU)**：執行算術（加、減、乘、除）和邏輯（與、或、非）運算。
    *   **控制單元 (Control Unit, CU)**：負責協調和控制計算機各部分的操作，解釋並執行指令。
    *   **暫存器 (Registers)**：CPU內部用於高速暫存資料的小型記憶體儲存區。
*   **記憶體 (Memory)**：用於儲存程式和資料。通常指的是隨機存取記憶體 (Random Access Memory, RAM)。
*   **輸入/輸出設備 (Input/Output Devices, I/O Devices)**：負責計算機與外部世界的溝通。
*   **匯流排 (Bus)**：一組電子通路，用於連接CPU、記憶體和I/O設備，傳輸資料、地址和控制訊號。

-----

#### 推導：指令執行週期

在馮紐曼架構下，CPU執行程式的過程是一個不斷重複的週期，稱為「指令執行週期 (Instruction Cycle)」或「提取-解碼-執行週期 (Fetch-Decode-Execute Cycle)」：

1.  **提取 (Fetch)**：控制單元從記憶體中取出（提取）下一條指令。
2.  **解碼 (Decode)**：控制單元解釋（解碼）這條指令的含義，確定需要執行什麼操作，以及操作的資料在哪裡。
3.  **執行 (Execute)**：算術邏輯單元執行指令所指定的運算（例如，加法、資料移動）。
4.  **寫回 (Write-back) / 儲存 (Store)**：將執行結果寫回記憶體或暫存器。

這個週期不斷重複，直到程式結束或電源關閉。

-----

#### 典型例子：一個簡單的加法指令執行

想像我們要執行一個簡單的指令：`R1 = R2 + R3` （將暫存器R2和R3的內容相加，結果存入R1）。

1.  **提取**：控制單元從記憶體中取出 `R1 = R2 + R3` 這條指令。
2.  **解碼**：控制單元辨識出這是一個加法操作，並且涉及暫存器R1、R2、R3。它會指示ALU準備執行加法，並從R2和R3中讀取數據。
3.  **執行**：ALU從R2和R3中獲取資料，執行加法運算。
4.  **寫回**：ALU將計算結果寫回暫存器R1。

-----

#### 與相鄰概念的關聯

馮紐曼架構是理解任何計算機硬體運作的基礎。它直接影響了計算機的性能、設計複雜度和程式語言的發展。例如，現代的多核心處理器、快取記憶體等都是為了優化馮紐曼架構下資料存取瓶頸（稱為「馮紐曼瓶頸」）而發展出來的。

### 3. 資料的表示：位元與位元組

#### 核心概念：二進制與數位化

計算機內部的一切資訊，無論是數字、文字、圖片或聲音，最終都必須轉換成二進制形式來儲存和處理。二進制系統只使用兩個符號：0 和 1。

*   **位元 (Bit)**：是計算機中最小的資料單位，代表一個二進制數字 (0 或 1)。它來自 "Binary Digit" 的縮寫。
*   **位元組 (Byte)**：通常是 8 個位元組成的一個單位。這是資料儲存和傳輸的基本單位。一個位元組可以表示 $2^8 = 256$ 種不同的狀態，足夠表示一個英文字母或一個小寫數字。

其他常見的資料單位：

*   1 KB (Kilobyte) = $1024$ Bytes
*   1 MB (Megabyte) = $1024$ KB
*   1 GB (Gigabyte) = $1024$ MB
*   1 TB (Terabyte) = $1024$ GB

注意：在某些儲存設備的標示中，廠商可能使用 1 KB = 1000 Bytes 的十進制標準，這導致實際容量略小於標示。但計算機科學領域通常採用 $2^{10}$ 的二進制標準。

-----

#### 典型例子與轉換：數字與文字的表示

**1. 數字的二進制表示**

*   **十進制到二進制轉換**：採用「除以2取餘數法」，將餘數倒序排列。

    **例子：將十進制數 $13$ 轉換為二進制。**
    $13 \div 2 = 6 \dots 1$
    $6 \div 2 = 3 \dots 0$
    $3 \div 2 = 1 \dots 1$
    $1 \div 2 = 0 \dots 1$
    將餘數從下往上排列，得到 $13_{10} = 1101_2$。

*   **二進制到十進制轉換**：將每個位元乘以其對應的 $2$ 的冪次，然後相加。

    **例子：將二進制數 $1101_2$ 轉換為十進制。**
    $1 \times 2^3 + 1 \times 2^2 + 0 \times 2^1 + 1 \times 2^0$
    $= 1 \times 8 + 1 \times 4 + 0 \times 2 + 1 \times 1$
    $= 8 + 4 + 0 + 1$
    $= 13_{10}$

**2. 文字的二進制表示**

文字在計算機中通常透過字符編碼標準轉換為數字，再由數字轉換為二進制。最常見的編碼標準包括：

*   **ASCII (American Standard Code for Information Interchange)**：使用 7 位元或 8 位元表示英文字母、數字和符號。例如，字母 'A' 的ASCII碼是 $65_{10}$，其二進制表示為 $01000001_2$。
*   **Unicode (萬國碼)**：為了解決ASCII無法表示多國語言的問題而設計，它提供了更大的字符集，如UTF-8、UTF-16等。每個字符可能佔用 1 到 4 個位元組。例如，中文字符會使用 Unicode 編碼。

-----

#### 與相鄰概念的關聯

資料的二進制表示是計算機內部所有運作的基礎。CPU的所有運算、記憶體的儲存、網路的傳輸，都離不開位元和位元組。理解這些概念對於學習資料結構、網路通訊、編碼加密等都至關重要。

-----

### 4. 計算機硬體組成

計算機的硬體是其物理部分，看得見、摸得著的實體設備。它們協同工作，共同實現計算機的功能。

#### 核心概念：主要組件與功能

*   **中央處理器 (CPU)**：前面已詳述，是計算機的核心運算單元。它的性能（主頻、核心數、快取）直接決定了計算機的整體速度。
*   **記憶體 (Memory)**：
    *   **隨機存取記憶體 (Random Access Memory, RAM)**：電腦工作時用來暫時儲存資料、程式和執行結果的地方。斷電後資料會消失（易失性）。RAM 容量越大，電腦同時處理多任務的能力越強。
    *   **唯讀記憶體 (Read Only Memory, ROM)**：儲存了計算機啟動時所需的基本程式（如BIOS/UEFI），斷電後資料不會消失（非易失性）。
*   **儲存設備 (Storage Devices)**：
    *   **硬碟機 (Hard Disk Drive, HDD)**：傳統的機械式儲存設備，使用磁盤記錄資料，容量大，成本相對較低，但速度較慢。
    *   **固態硬碟 (Solid State Drive, SSD)**：使用快閃記憶體儲存資料，速度快，抗震性好，但成本較高。
    *   **其他**：USB隨身碟、SD卡、光碟機等。
*   **輸入設備 (Input Devices)**：
    *   **鍵盤 (Keyboard)**：輸入文字和指令。
    *   **滑鼠 (Mouse)**：控制螢幕上的指標，進行點擊、拖曳。
    *   **麥克風 (Microphone)**：輸入聲音。
    *   **掃描器 (Scanner)**：將紙本文件轉換為數位影像。
    *   **觸控螢幕 (Touchscreen)**：直接點擊螢幕進行操作。
*   **輸出設備 (Output Devices)**：
    *   **顯示器/螢幕 (Monitor/Screen)**：顯示影像和文字。
    *   **印表機 (Printer)**：列印文件。
    *   **喇叭 (Speakers)**：輸出聲音。
    *   **投影機 (Projector)**：將影像投射到大螢幕。
*   **主機板 (Motherboard)**：計算機的「骨架」，所有其他硬體組件都連接到主機板上，透過主機板上的匯流排進行通訊。
*   **顯示卡/圖形處理器 (Graphics Card/GPU)**：負責處理和輸出圖形影像，對於遊戲、影音編輯和人工智慧運算至關重要。
*   **網路卡 (Network Interface Card, NIC)**：使計算機能夠連接到網路。

-----

#### 典型例子：電腦的開機流程

當你按下電腦的電源按鈕時，一系列硬體組件會協同工作：

1.  **電源供應器**提供電力。
2.  **CPU** 開始執行主機板 **ROM** 中的 **BIOS/UEFI** 程式。
3.  BIOS/UEFI 執行自我檢測 (POST)，檢查基本硬體是否正常。
4.  BIOS/UEFI 接著尋找 **儲存設備** (如SSD/HDD) 中的作業系統。
5.  作業系統的核心部分被載入到 **RAM** 中。
6.  作業系統開始啟動，並管理其他硬體和軟體資源。
7.  最終，作業系統透過 **顯示卡** 和 **顯示器** 呈現出使用者介面，等待使用者透過 **鍵盤** 或 **滑鼠** 輸入指令。

-----

#### 與相鄰概念的關聯

硬體是計算機的物理基礎，而軟體則是在硬體之上運行的「靈魂」。沒有硬體，軟體無法執行；沒有軟體，硬體則是一堆無用的金屬。兩者是密不可分的共生關係。硬體技術的進步直接推動了軟體功能的發展，例如GPU的進化促進了深度學習的突破。

-----

### 5. 計算機軟體概觀

軟體是指令的集合，告訴硬體如何執行特定的任務。它分為兩大類：系統軟體和應用軟體。

#### 核心概念：系統軟體與應用軟體

*   **系統軟體 (System Software)**：
    *   負責管理和控制計算機硬體資源，提供一個平台供應用軟體運行。
    *   **作業系統 (Operating System, OS)**：是最重要的系統軟體，如 Windows、macOS、Linux、Android、iOS。它管理記憶體、行程、檔案系統、I/O設備，並提供使用者介面。
    *   **驅動程式 (Device Drivers)**：允許作業系統與特定的硬體設備（如印表機、顯示卡）進行通訊。
    *   **公用程式 (Utility Programs)**：輔助性工具，如防毒軟體、磁碟整理工具、壓縮軟體。
*   **應用軟體 (Application Software)**：
    *   為特定任務或使用者需求設計的程式。它們依賴於作業系統才能運行。
    *   **文書處理軟體**：Microsoft Word, Google Docs。
    *   **網頁瀏覽器**：Google Chrome, Mozilla Firefox, Microsoft Edge。
    *   **媒體播放器**：VLC Media Player, Spotify。
    *   **遊戲**：Minecraft, Cyberpunk 2077。
    *   **開發工具**：Visual Studio Code, Eclipse。

-----

#### 典型例子：軟體如何讓電腦可用

想像一台沒有安裝作業系統的電腦，它就像一個空盒子。你無法用它來上網、寫文件或玩遊戲。

1.  **作業系統 (Windows)**：當你安裝 Windows 後，它會管理你的 CPU、RAM、硬碟、顯示器等硬體，並提供一個圖形化介面，讓你可以用滑鼠點擊圖標。
2.  **應用程式 (Google Chrome)**：你安裝 Chrome 瀏覽器後，它會利用 Windows 提供的網路功能去存取網站，利用顯示功能來顯示網頁內容。Chrome 本身就是一套龐大的指令集，讓電腦完成「瀏覽網頁」這個特定任務。
3.  **程式語言 (Python)**：如果你想自己開發一個小型遊戲，你需要使用像 Python 這樣的程式語言來編寫指令。這些指令會被「編譯」或「解釋」成機器碼，最終交給 CPU 執行。

-----

#### 與相鄰概念的關聯

軟體是人機互動的橋樑。它將使用者的意圖轉換為硬體可以理解的指令。沒有軟體，再強大的硬體也無法發揮作用。軟體工程、程式設計、資料庫管理等領域都圍繞著如何高效、安全、可靠地開發和維護軟體而展開。

-----

### 常見錯誤與澄清

#### 1. 混淆「記憶體 (RAM)」與「儲存設備 (硬碟/SSD)」

*   **常見錯誤**：許多初學者會將「記憶體」和「硬碟」視為同一回事，或認為記憶體越大，儲存的檔案就越多。
*   **澄清**：
    *   **記憶體 (RAM)**：是**臨時工作區**。電腦執行程式時，程式本身和它正在處理的資料會被載入到 RAM 中。RAM 的特點是**速度快、容量相對小、斷電後資料消失**。想像成你的書桌，用來放置正在看的書和筆記。
    *   **儲存設備 (HDD/SSD)**：是**長期儲存區**。所有的作業系統、程式、文件、照片等都永久儲存在這裡。儲存設備的特點是**容量大、速度相對慢（SSD比HDD快）、斷電後資料不消失**。想像成你的書架或檔案櫃，用來長期保存所有資料。
*   **舉例**：當你開啟一個大型軟體（如Photoshop），它會從你的SSD載入到RAM中運行。如果你的RAM不足，電腦就會變慢，因為它必須頻繁地將RAM中的資料交換到SSD上，這個過程稱為「虛擬記憶體」或「頁面交換」。

-----

#### 2. 誤解「CPU核心數」與「程式執行」的關係

*   **常見錯誤**：認為 CPU 有兩個核心，程式就會快兩倍。
*   **澄清**：
    *   CPU核心數確實代表了處理器可以同時執行多少個指令序列（執行緒）。
    *   **單核**：一次只能處理一個任務。
    *   **多核**：可以同時處理多個任務（或一個任務的不同部分）。
    *   但程式能否利用多核的優勢，**取決於程式本身是否設計成可以「平行處理 (Parallel Processing)」**。如果一個程式是單執行緒的，無論你的CPU有多少核心，它都只會利用其中一個核心。只有當程式被設計成可以將任務分解成多個獨立部分，並讓這些部分同時在不同核心上執行時，多核CPU才能顯著提升性能。
*   **舉例**：瀏覽網頁（多個分頁、多個下載）通常能很好地利用多核。而某些老舊的單機遊戲，可能就只會利用一個核心。

### 小練習

#### 練習 1：二進制轉換

請將以下數字進行二進制與十進制的轉換：

1.  將十進制數 $25_{10}$ 轉換為二進制數。
2.  將二進制數 $10110_2$ 轉換為十進制數。

**詳解**：

1.  **十進制 $25_{10}$ 轉換為二進制：**
    *   $25 \div 2 = 12 \dots 1$
    *   $12 \div 2 = 6 \dots 0$
    *   $6 \div 2 = 3 \dots 0$
    *   $3 \div 2 = 1 \dots 1$
    *   $1 \div 2 = 0 \dots 1$
    將餘數倒序排列，得到 $11001_2$。
    **答案：** $25_{10} = 11001_2$

2.  **二進制 $10110_2$ 轉換為十進制：**
    *   從右往左，每個位元乘以 $2$ 的冪次，然後相加。
    *   $0 \times 2^0 = 0 \times 1 = 0$
    *   $1 \times 2^1 = 1 \times 2 = 2$
    *   $1 \times 2^2 = 1 \times 4 = 4$
    *   $0 \times 2^3 = 0 \times 8 = 0$
    *   $1 \times 2^4 = 1 \times 16 = 16$
    *   將結果相加：$0 + 2 + 4 + 0 + 16 = 22$
    **答案：** $10110_2 = 22_{10}$

-----

#### 練習 2：資料單位計算

如果一張未壓縮的彩色圖片，其解析度為 1920x1080 像素，每個像素使用 24 位元 (bit) 來表示顏色深度。請問這張圖片的檔案大小是多少 MB？

**詳解**：

1.  **計算總像素數**：
    *   總像素 = 寬度 $\times$ 高度 = $1920 \times 1080 = 2073600$ 像素。

2.  **計算總位元數 (bits)**：
    *   總位元數 = 總像素數 $\times$ 每個像素的位元數 = $2073600 \times 24$ 位元 = $49766400$ 位元。

3.  **將位元轉換為位元組 (Bytes)**：
    *   由於 1 位元組 (Byte) = 8 位元 (bits)，
    *   總位元組數 = 總位元數 $\div 8 = 49766400 \div 8 = 6220800$ 位元組。

4.  **將位元組轉換為 MB (Megabytes)**：
    *   由於 1 MB = $1024$ KB，1 KB = $1024$ Bytes，所以 1 MB = $1024 \times 1024 = 1048576$ Bytes。
    *   檔案大小 (MB) = 總位元組數 $\div 1048576 = 6220800 \div 1048576 \approx 5.932$ MB。

    **答案：** 約為 $5.932$ MB。
