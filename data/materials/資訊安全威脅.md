主題：1-3 資訊安全威脅

#### 1. 核心概念：什麼是資訊安全威脅？

*   **定義：** 資訊安全威脅（Information Security Threat）是指任何可能對資訊系統、資料或網路造成損害、導致資訊洩露、破壞完整性或可用性，或阻礙正常運作的潛在事件或行為。這些事件可能是惡意的、意外的，或是由環境因素引起的。

*   **典型例子：**
    *   駭客嘗試入侵公司伺服器。
    *   員工不小心點擊了帶有惡意軟體的連結。
    *   機房因地震或火災導致設備損壞。

*   **與相鄰概念的關聯：**
    在資訊安全領域，有幾個緊密相關的概念：
    *   **威脅 (Threat):** 潛在的危害。它是「可能導致負面結果的事件」。例如：駭客、惡意軟體、自然災害。
    *   **弱點 (Vulnerability):** 系統、軟體、硬體或流程中可被威脅利用的缺陷或漏洞。它是「威脅能夠利用的通道」。例如：未打補丁的作業系統、弱密碼、配置錯誤。
    *   **攻擊 (Attack):** 威脅利用弱點以實施危害的具體行為或行動。它是「威脅利用弱點的實際過程」。例如：網路釣魚、SQL 隱碼攻擊。
    *   **風險 (Risk):** 威脅利用弱點導致損失的可能性及其後果。它表示「一旦發生威脅攻擊，可能造成的損失」。風險通常可表示為：$$ \text{風險} = \text{威脅} \times \text{弱點} \times \text{衝擊} $$
    *   **例子：** 一名駭客（威脅）發現某網站程式碼存在 SQL 注入漏洞（弱點），他利用這個漏洞成功竊取了資料庫內容（攻擊），導致網站用戶個資外洩（風險）。

-----

#### 2. 資訊安全威脅的典型類別與例子

我們會將資訊安全威脅分為幾大主要類別，每類提供核心觀念、具體例子與其常見行為模式。

### 2.1 惡意軟體 (Malware)

*   **核心觀念：** 旨在破壞、竊取數據、未經授權存取系統或執行其他惡意操作的軟體。Malware 是 "Malicious Software" 的縮寫。

*   **典型例子與轉換/推導：**
    *   **病毒 (Virus):** 依附於合法程式，透過使用者執行動作（如開啟檔案）感染並自我複製，從而傳播。
        *   例：巨集病毒感染 Word 文件，當文件開啟時，病毒碼便會執行，並嘗試感染其他文件。
    *   **蠕蟲 (Worm):** 獨立的惡意程式，透過網路自我複製並傳播，無需宿主程式或使用者操作。
        *   例：2000 年的「愛蟲病毒 (ILOVEYOU)」，透過電子郵件附件傳播，感染後會自動發送給通訊錄中的所有聯絡人，並刪除特定檔案。
    *   **木馬程式 (Trojan Horse):** 偽裝成合法或有用程式，欺騙使用者安裝與執行，一旦執行便在後台執行惡意行為，如開啟後門、竊取資訊。
        *   例：從非官方管道下載的「免費破解版遊戲」，實際卻捆綁了竊取銀行帳戶資訊的木馬程式。
    *   **勒索軟體 (Ransomware):** 綁架使用者檔案或系統，透過加密等方式使其無法存取，並要求支付贖金才能解密或恢復存取。
        *   例：WannaCry 勒索病毒，利用 Windows 系統漏洞，加密使用者硬碟上的檔案並要求比特幣贖金，否則將永久鎖定檔案。
    *   **間諜軟體 (Spyware):** 在使用者不知情下收集個人資訊、瀏覽行為或鍵盤輸入等數據，並傳送給第三方。
        *   例：在瀏覽器中安裝的惡意擴充功能，記錄你的所有瀏覽歷史、帳號密碼輸入，並發送給攻擊者。
    *   **廣告軟體 (Adware):** 強制顯示廣告，通常與免費軟體捆綁安裝，可能修改瀏覽器設定或收集使用者數據。
        *   例：電腦安裝某應用程式後，網頁瀏覽器首頁被自動修改，且不斷跳出與瀏覽內容無關的彈出式廣告。

*   **與相鄰概念的關聯：** 惡意軟體常透過「社會工程」技術散播，並利用系統的「弱點」來植入或執行。它主要影響資訊的「機密性」、「完整性」和「可用性」。

-----

### 2.2 社會工程 (Social Engineering)

*   **核心觀念：** 透過欺騙、誘騙、恐嚇等非技術手段，操縱人類心理，使其洩露機密資訊、提供存取權限或執行某些行動的攻擊。它是利用人性的弱點而非系統漏洞。

*   **典型例子與轉換/推導：**
    *   **網路釣魚 (Phishing):** 偽裝成可信任的實體（如銀行、政府機關、知名企業），透過電子郵件、簡訊、即時通訊等方式誘騙目標洩露個人資料（帳號、密碼、信用卡號）或點擊惡意連結。
        *   例：收到一封假冒銀行發送的電子郵件，聲稱您的帳戶有異常，要求您點擊連結登入進行驗證。該連結實為模仿銀行官網的惡意網站。
    *   **語音釣魚 (Vishing):** 透過電話進行的釣魚攻擊。攻擊者假冒客服人員、公務員或技術支援人員。
        *   例：假冒電信公司客服來電，聲稱您的電話費欠繳，若不立即提供銀行卡號處理將被停話。
    *   **簡訊釣魚 (Smishing):** 透過手機簡訊 (SMS) 進行的釣魚攻擊。
        *   例：收到一則簡訊，聲稱您包裹的運費未繳，附上一個不明連結要求您支付，點擊後可能被安裝惡意程式或竊取個資。
    *   **預謀詐騙 (Pretexting):** 攻擊者事先編造好一個「故事情節」或「謊言」，以獲取受害者的信任，從而誘騙其提供資訊或採取行動。
        *   例：攻擊者假扮成 IT 部門人員，聯繫受害者稱需要他們的密碼來「測試系統」以解決「技術問題」。
    *   **誘餌攻擊 (Baiting):** 透過提供誘惑性物品（如免費軟體、音樂、電影，或留下看似重要的 USB 隨身碟）來誘騙受害者。
        *   例：在辦公室或公共場所丟下一個貼有「公司薪資表」或「機密專案」標籤的 USB 隨身碟，期望好奇者會撿起並插入電腦，隨身碟內則含有惡意軟體。
    *   **尾隨 (Tailgating / Piggybacking):** 跟隨合法授權的人員進入受限區域，利用他人的合法進入權限。
        *   例：趁員工刷卡進入大樓時，假裝講電話或手上提滿物品，要求員工幫忙開門，從而進入管制區域。

*   **與相鄰概念的關聯：** 社會工程是許多「惡意軟體」和「資料竊取」攻擊的常見初始環節，它利用的是「人」這個最弱的環節，主要影響資訊的「機密性」。

### 2.3 阻斷服務攻擊 (Denial of Service, DoS / DDoS)

*   **核心觀念：** 透過大量無效請求或惡意流量，癱瘓目標伺服器、網路或應用程式，使其無法提供正常服務給合法使用者。

*   **典型例子與轉換/推導：**
    *   **DoS (阻斷服務):** 單一攻擊源對目標發動攻擊。由於攻擊源單一，相對容易被追蹤和阻斷。
        *   例：Ping of Death，攻擊者發送超大尺寸的 Ping 封包導致目標系統崩潰或網路癱瘓。
    *   **DDoS (分散式阻斷服務):** 多個攻擊源（通常是已被感染的「殭屍網路」中的電腦或物聯網設備）同時對目標發動攻擊，產生海量流量。由於攻擊源分散且數量龐大，難以追蹤和防禦。
        *   例：利用殭屍網路對某電商網站發動 SYN Flood 攻擊，發送大量的 TCP SYN 請求，耗盡網站伺服器的連接資源，導致網站無法回應正常使用者的請求。
    *   **與相鄰概念的關聯：** DDoS 攻擊常利用被「惡意軟體」感染的電腦組成的「殭屍網路」，攻擊的目的是破壞資訊的「可用性」。

-----

### 2.4 資料外洩與竊取 (Data Breach & Theft)

*   **核心觀念：** 未經授權地存取、複製、傳輸、洩露或使用敏感資訊的事件。這類威脅直接針對資料的機密性。

*   **典型例子與轉換/推導：**
    *   **SQL 隱碼攻擊 (SQL Injection):** 攻擊者在網頁輸入框中輸入惡意 SQL 語句，繞過驗證、讀取資料庫敏感資訊，甚至篡改資料。
        *   例：在使用者名稱輸入 `admin' OR '1'='1`，使後端查詢語句 `SELECT * FROM users WHERE username='admin' OR '1'='1' AND password='...'` 永遠為真，繞過密碼驗證而成功登入，或竊取資料庫內容。
    *   **暴力破解 (Brute Force Attack):** 攻擊者嘗試所有可能的密碼組合或常見密碼列表，來猜測使用者帳號的密碼。
        *   例：使用自動化工具，對某網站的登錄頁面，以每秒數千次的頻率嘗試從 `000000` 到 `999999` 的所有6位數密碼，直到猜中為止。
    *   **憑證填充 (Credential Stuffing):** 攻擊者利用已在其他網站洩露的用戶名和密碼（因為許多使用者在不同網站使用相同的憑證），在目標網站上嘗試登錄。
        *   例：攻擊者從一個論壇的資料外洩事件中獲取了一批用戶名和密碼，然後嘗試用這些憑證登錄熱門的電子商務網站或社交媒體，以竊取帳號。
    *   **內部威脅 (Insider Threat):** 組織內部人員（包括現任或離職員工、承包商、合作夥伴等）因惡意意圖、疏忽或被脅迫，而洩露資訊、破壞系統或導致資料遺失。
        *   例：一位不滿的員工在離職前，利用其原有的權限將公司客戶資料庫複製帶走，打算轉賣給競爭對手。
    *   **物理竊取：** 直接偷竊含有敏感資料的實體設備，如筆記型電腦、伺服器、硬碟或儲存媒體。
        *   例：公司筆記型電腦在出差途中被竊，其中包含大量未加密的客戶資訊。

*   **與相鄰概念的關聯：** 資料外洩常是「弱點」未被修補或「社會工程」成功利用的結果，其目標是破壞資訊的「機密性」和「完整性」。

-----

### 2.5 系統漏洞與配置錯誤 (System Vulnerabilities & Misconfigurations)

*   **核心觀念：** 軟體、硬體或網路設定中的缺陷或不當設置，這些缺陷本身不是攻擊，但它們為攻擊者提供了可利用的機會。

*   **典型例子與轉換/推導：**
    *   **未修補的軟體漏洞 (Unpatched Software Vulnerabilities):** 未安裝作業系統、應用程式、韌體或網路設備的安全更新，導致已知的弱點暴露。
        *   例：微軟的「永恆之藍 (EternalBlue)」漏洞，許多組織因未及時更新 Windows 系統補丁，被勒索軟體 WannaCry 廣泛利用入侵。
    *   **預設或弱密碼 (Default/Weak Passwords):** 使用設備或軟體出廠時的預設密碼，或使用容易猜測的密碼（如 `123456`、`password`）。
        *   例：公司路由器仍使用 `admin/admin` 作為登入憑證，攻擊者可輕易登入並修改網路設定，甚至植入惡意韌體。
    *   **不安全的網路配置 (Insecure Network Configurations):** 開啟不必要的網路服務、防火牆規則配置不當、缺乏網路分段等。
        *   例：伺服器不必要地開放了遠端桌面服務 (RDP) 端口，且沒有設置強密碼或限制 IP 存取，攻擊者可透過暴力破解嘗試登入。
    *   **權限配置不當 (Improper Permissions):** 檔案、資料夾或系統資源的存取權限設置過於寬鬆，導致非授權使用者也能讀取、寫入或執行敏感資料。
        *   例：某個存放公司財務報表的資料夾，設定了 `everyone` 皆可讀寫的權限，導致任何內部員工都能隨意修改或竊取。

*   **與相鄰概念的關聯：** 這些「弱點」本身不是「威脅」，但它們是「威脅」實現「攻擊」的先決條件。修補這些弱點是防禦策略的基礎。

#### 3. 與相鄰概念的關聯：CIA 三要素

資訊安全威脅通常會針對資訊安全的「機密性 (Confidentiality)」、「完整性 (Integrity)」和「可用性 (Availability)」這三要素（統稱為 CIA 三要素）之一或多個進行破壞。

*   **機密性 (Confidentiality):**
    *   **核心觀念：** 確保資訊只被授權人員、實體或過程存取。防止資訊被未經授權的個人、團體或實體讀取或洩露。
    *   **威脅目標：** 惡意軟體（間諜軟體）、社會工程（網路釣魚）、資料外洩與竊取（SQL隱碼、內部威脅、暴力破解）。
    *   **例子：** 信用卡號被竊取、公司機密文件被未授權的員工複製。

*   **完整性 (Integrity):**
    *   **核心觀念：** 確保資訊在儲存、處理和傳輸過程中是準確、完整且未被未經授權地修改或破壞的。
    *   **威脅目標：** 惡意軟體（病毒、勒索軟體導致文件被修改或無法使用）、未經授權的修改（駭客篡改網站內容、資料庫數據）、系統漏洞（允許資料被竄改）。
    *   **例子：** 銀行帳戶金額被駭客篡改、公司網站首頁被植入不雅內容。

*   **可用性 (Availability):**
    *   **核心觀念：** 確保授權使用者在需要時，可以及時且可靠地存取資訊和相關的資訊系統與資源。
    *   **威脅目標：** 阻斷服務攻擊 (DoS/DDoS)、惡意軟體（勒索軟體使系統無法運作、病毒導致系統崩潰）、物理威脅（伺服器被竊取或損壞）、環境威脅（停電、地震）。
    *   **例子：** 電商網站因 DDoS 攻擊而癱瘓、公司伺服器因硬體故障導致服務中斷。

#### 4. 常見錯誤與澄清

*   **誤解一：威脅、弱點、風險傻傻分不清楚。**
    *   **澄清：** 這三者是資訊安全評估中的關鍵概念，彼此關聯但定義不同。
        *   **威脅 (Threat):** 潛在的危害源。例如：駭客、惡意程式、自然災害。
        *   **弱點 (Vulnerability):** 系統或流程中的缺陷，讓威脅有機可乘。例如：未打補丁的軟體、弱密碼。
        *   **風險 (Risk):** 威脅利用弱點導致損失的可能性及後果。
        *   **例子：** 地震（威脅）導致你家老舊建築的牆壁出現裂縫（弱點），使得在地震中房屋倒塌、人員受傷的可能性（風險）大大增加。

*   **誤解二：只有大型企業或政府機關才會成為攻擊目標。**
    *   **澄清：** 任何連上網路的個人、小型企業、中型組織都可能是攻擊目標。許多攻擊（如惡意軟體散播、DDoS 攻擊）是自動化掃描網路尋找弱點，而非針對特定目標。小型企業可能因防禦資源較少、安全意識不足，反而更容易得手。攻擊者也可能將小型組織作為跳板，進而攻擊其合作的大型企業。

*   **誤解三：只要安裝防毒軟體就萬無一失。**
    *   **澄清：** 防毒軟體是重要的第一道防線，但它只能防禦已知的惡意軟體，並且需要保持更新。面對零時差攻擊 (Zero-day Attack)、新型的變種惡意軟體、社會工程攻擊、內部威脅、配置錯誤等，單純的防毒軟體作用有限。資訊安全是多層次的，需要綜合防禦策略，包括防火牆、入侵檢測系統、定期備份、員工安全意識培訓、強密碼政策等。

#### 5. 小練習（附詳解）

##### 練習一：情境判斷題

請閱讀以下情境，並判斷其主要涉及的資訊安全威脅類別，並簡要說明原因。

1.  **情境：** 小陳收到一封電子郵件，郵件標題是「恭喜您中獎！點擊領取獎金」，發件人看似是某知名百貨公司。小陳點擊連結後，被要求輸入姓名、地址和銀行卡號碼以領取獎金。
2.  **情境：** 某新聞網站突然無法存取，其 IT 部門發現伺服器收到異常大量的連接請求，導致網站回應速度極慢最終崩潰，而且這些請求來自全球數十萬個不同的 IP 位址。
3.  **情境：** 阿華在網路上下載了一款聲稱可免費加速網路的工具。安裝後，他的電腦速度並沒有明顯變快，反而發現每次開機時，都會有一些奇怪的程式在背景執行，且不定時會跳出一些不認識的視窗。
4.  **情境：** 某公司的一名系統管理員因對公司待遇不滿，在離職前悄悄地在公司的核心資料庫中植入了一段惡意程式碼，該程式碼會在一個月後自動刪除部分重要客戶資料。

##### 詳解一：

1.  **威脅類別：** 社會工程 (特別是網路釣魚 Phishing)。
    *   **原因：** 攻擊者偽裝成知名百貨公司（可信任實體），透過電子郵件誘騙小陳點擊惡意連結並在假網站上輸入敏感個人資料和銀行卡號，這是典型的利用人類心理弱點進行欺騙的行為。
2.  **威脅類別：** 分散式阻斷服務攻擊 (DDoS)。
    *   **原因：** 來自「全球數十萬個不同的 IP 位址」發送「異常大量的連接請求」，旨在「耗盡伺服器資源」並導致「網站無法存取」，這些都是 DDoS 攻擊的典型特徵。
3.  **威脅類別：** 惡意軟體 (可能是木馬程式 Trojan Horse 或間諜軟體 Spyware)。
    *   **原因：** 軟體「聲稱免費加速網路」作為誘餌，安裝後卻在「背景執行奇怪程式」且「不定時跳出不認識的視窗」，暗示該軟體可能在使用者不知情的情況下執行惡意行為，如監控、植入其他惡意程式或顯示廣告。
4.  **威脅類別：** 內部威脅 (Insider Threat) 與資料完整性破壞。
    *   **原因：** 該系統管理員是「組織內部人員」，利用其權限「植入惡意程式碼」並導致「重要客戶資料被刪除」，這屬於惡意內部人員行為，直接破壞了資料的完整性和可用性。

##### 練習二：概念配對題

請將左側的資訊安全威脅情境，與右側其主要攻擊目標的 CIA 三要素進行配對。

| 資訊安全威脅情境                                | 主要攻擊目標的 CIA 三要素 |
| :---------------------------------------------- | :-------------------------- |
| 1. 公司會計系統的數據被駭客未經授權地修改         | A. 機密性 (Confidentiality) |
| 2. 勒索軟體加密了學校所有電腦的檔案，導致無法使用 | B. 完整性 (Integrity)       |
| 3. 一名員工將機密客戶名單洩露給競爭對手         | C. 可用性 (Availability)    |
| 4. 網站伺服器因軟體漏洞頻繁崩潰，無法提供服務   |                             |

##### 詳解二：

1.  **公司會計系統的數據被駭客未經授權地修改** → **B. 完整性 (Integrity)**
    *   **原因：** 數據被未經授權地修改，破壞了其準確性和完整性。
2.  **勒索軟體加密了學校所有電腦的檔案，導致無法使用** → **C. 可用性 (Availability)** (同時也影響完整性)
    *   **原因：** 檔案被加密後，導致合法使用者無法存取和使用，嚴重影響了資訊的可用性。雖然檔案內容被加密修改也影響了原始可讀的完整性，但此情境下主要衝擊是無法「使用」。
3.  **一名員工將機密客戶名單洩露給競爭對手** → **A. 機密性 (Confidentiality)**
    *   **原因：** 機密資訊被未經授權的人員（競爭對手）存取，違反了資訊的機密性。
4.  **網站伺服器因軟體漏洞頻繁崩潰，無法提供服務** → **C. 可用性 (Availability)**
    *   **原因：** 網站無法正常運作，合法使用者無法存取其服務，違反了資訊的可用性。

*   **網路安全基本概念：** 學習更多關於防火牆、入侵檢測系統 (IDS)、入侵防禦系統 (IPS) 等常見的網路安全防禦技術及其運作原理。
