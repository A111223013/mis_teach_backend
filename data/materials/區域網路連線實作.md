# 區域網路連線實作

## ### 1. 核心概念與定義

#### #### 1.1 什麼是區域網路 (Local Area Network, LAN)

**定義/核心觀念：**
區域網路（LAN）是指在一個相對較小的地理範圍內（例如：家庭、辦公室、學校建築物等）將多台電腦或其他網路裝置（如印表機、伺服器）互相連接起來的網路。其主要目的是實現資源共享（文件、印表機、網際網路連線等）和設備之間的通訊。LAN 的傳輸速度通常比廣域網路（WAN）快，且延遲較低。

**例子：**
*   您家中的路由器連接了您的電腦、手機、智慧電視和網路印表機，這就是一個典型的家庭區域網路。
*   辦公室裡多台電腦透過交換器連接，並共享一台伺服器和印表機，這是一個典型的企業區域網路。

**與相鄰概念的關聯：**
*   **廣域網路 (WAN)：** LAN 是構成 WAN 的基礎單元。WAN 負責連接地理位置上較遠的 LAN，例如網際網路本身就是一個巨型 WAN。
*   **無線區域網路 (WLAN)：** WLAN 是 LAN 的一種特殊形式，它使用無線電波（如 Wi-Fi）而非實體線材來連接裝置。兩者在邏輯功能上相似，但在物理傳輸方式上有所區別。

-----

#### #### 1.2 區域網路的基礎構成要素

**定義/核心觀念：**
建立一個區域網路需要以下幾種核心元件：

1.  **網路介面卡 (Network Interface Card, NIC / 網卡)：** 它是電腦或其他裝置連接到網路的硬體介面。它將電腦的數位資料轉換為可以在網路上傳輸的電氣或光學訊號。
2.  **網路線材 (Network Cables)：** 用於連接網路裝置，最常見的是乙太網路線 (Ethernet Cable)，標準接頭為 RJ45。常見規格有 Cat5e, Cat6, Cat6a 等，影響傳輸速度和距離。
3.  **網路設備：**
    *   **集線器 (Hub)：** 一種較舊的網路設備。它接收到任何一個連接埠的訊號後，會將該訊號無差別地廣播到所有其他連接埠。這會導致網路碰撞和效率低下，目前已很少使用。
    *   **交換器 (Switch)：** 比 Hub 更智慧。它能學習連接在每個埠上的裝置的 MAC 位址，並將資料幀精準地轉發到目標裝置所在的埠，而非廣播。這是現代 LAN 的核心設備。
    *   **路由器 (Router)：** 主要用於連接不同的網路（例如 LAN 與 WAN），它透過 IP 位址來轉發資料包，並負責處理網路層（OSI 模型第三層）的路由決策。家庭路由器通常整合了交換器、Wi-Fi 存取點和防火牆功能。
4.  **IP 位址 (IP Address)：** 網路裝置在網路上的唯一識別碼，用於在網路層識別和定位裝置。分為 IPv4 和 IPv6。
5.  **子網路遮罩 (Subnet Mask)：** 用於區分 IP 位址中屬於網路部分和主機部分，以判斷兩個 IP 位址是否在同一個區域網路內。
6.  **預設閘道 (Default Gateway)：** 當網路裝置要傳送資料到自身區域網路以外的目標時，資料包會被送往預設閘道，通常是路由器的 IP 位址。

**例子：**
*   您的電腦內部通常有一張內建的乙太網路卡或 Wi-Fi 網卡。
*   一條 Cat6 網路線連接您的電腦到交換器的一個 RJ45 埠。
*   交換器有 8 個 RJ45 埠，可以連接 8 台電腦或其他網路設備。
*   您的電腦 IP 位址可能是 `192.168.1.100`，子網路遮罩 `255.255.255.0`，預設閘道 `192.168.1.1` (路由器的 IP)。

**與相鄰概念的關聯：**
這些元件共同協作，從物理層到網路層，實現了資料在區域網路內的傳輸和設備之間的通訊。網卡和網路線是物理層（L1）的實現；交換器在資料連結層（L2）運作；路由器和 IP 位址在網路層（L3）運作。

## ### 2. 典型實作步驟與情境

#### #### 2.1 準備硬體與物理連線

**定義/核心觀念：**
在進行任何軟體設定之前，確保所有必要的硬體設備都已準備就緒，並且按照正確的方式連接。

**實作步驟：**

1.  **清點所需硬體：**
    *   至少兩台具備乙太網路卡的電腦。
    *   一台交換器 (Switch)。
    *   足夠數量的乙太網路線 (Cat5e 或 Cat6，直通線)，每台電腦一台交換器至少需要一條。
    *   電源線，用於交換器。

2.  **連接電源：** 將交換器插上電源。通常交換器啟動後會有指示燈亮起，表示電源正常。

3.  **連接網路線：**
    *   將每台電腦的乙太網路埠（RJ45 孔）與交換器上任意一個可用的 RJ45 埠連接起來。
    *   確保 RJ45 接頭插入時聽到「咔噠」聲，表示連接牢固。
    *   檢查交換器上對應連接埠的指示燈，通常會顯示綠燈或閃爍，表示連線正常且有資料活動。

**典型例子：**
假設您有兩台電腦 (PC1, PC2) 和一個 8 埠交換器。
*   將 PC1 的網卡埠用一條網路線連接到交換器的埠 1。
*   將 PC2 的網卡埠用另一條網路線連接到交換器的埠 2。
*   交換器插上電源。

**與相鄰概念的關聯：**
這部分是 OSI 模型中的物理層 (L1) 工作。只有物理連接建立成功，後續的資料連結層、網路層設定才能發揮作用。

-----

#### #### 2.2 軟體配置：IP 位址設定

**定義/核心觀念：**
物理連線完成後，每台網路裝置都需要一個 IP 位址才能在網路上被識別和通訊。IP 位址設定通常有兩種方式：自動取得 (DHCP) 或手動設定 (固定 IP)。在實作區域網路連線時，手動設定固定 IP 有助於理解 IP 運作原理。

**實作步驟 (手動設定固定 IP)：**
以 Windows 10/11 為例：

1.  **打開網路和網際網路設定：**
    *   右鍵點擊任務列上的網路圖示 (通常是電腦或 Wi-Fi 符號)。
    *   選擇「網路和網際網路設定」。
2.  **進入網路介面卡選項：**
    *   在左側選單中選擇「乙太網路」。
    *   點擊右側的「變更介面卡選項」或「變更介面卡設定」。
3.  **編輯網路介面卡內容：**
    *   找到您正在使用的乙太網路介面卡（通常名稱為「乙太網路」或「本機連線」），右鍵點擊它。
    *   選擇「內容」。
4.  **設定 IPv4 屬性：**
    *   在彈出的視窗中，找到並選中「網際網路通訊協定第 4 版 (TCP/IPv4)」。
    *   點擊「內容」。
5.  **輸入 IP 位址資訊：**
    *   選擇「使用下列 IP 位址」。
    *   **IP 位址：** 為每台電腦設定一個唯一的 IP 位址。例如：
        *   PC1: `192.168.1.10`
        *   PC2: `192.168.1.11`
        *   ...
        *   **注意：** 確保所有電腦的 IP 位址前三段相同，第四段不同。
    *   **子網路遮罩：** 統一設定為 `255.255.255.0`。
    *   **預設閘道：** 在兩台電腦直連或僅透過交換器連線的簡單 LAN 環境中，若不需連外網，此項可以留空。若要連外網，則應填寫路由器的 IP 位址 (例如 `192.168.1.1`)。為了本練習的純 LAN 通訊，可先留空或填寫一個假想的 IP，只要不與其他裝置衝突即可，例如 `192.168.1.1`。
    *   **慣用 DNS 伺服器：** 可以留空，或填寫一個公共 DNS (例如 `8.8.8.8` Google DNS) 或路由器的 IP。對於純 LAN 內部通訊不是必需。
6.  **確認並關閉視窗：** 點擊「確定」關閉所有視窗以保存設定。

**典型例子：**
PC1 設定：
*   IP 位址: `192.168.1.10`
*   子網路遮罩: `255.255.255.0`
*   預設閘道: `192.168.1.1` (假想路由器 IP)
PC2 設定：
*   IP 位址: `192.168.1.11`
*   子網路遮罩: `255.255.255.0`
*   預設閘道: `192.168.1.1` (假想路由器 IP)

**與相鄰概念的關聯：**
IP 位址設定是 OSI 模型中網路層 (L3) 的核心配置。子網路遮罩決定了哪些 IP 位址在同一個邏輯網路中，而預設閘道則負責將資料包轉發到不同網路。DHCP (動態主機設定協定) 則是一種應用層協定，用於自動分發這些網路層配置資訊。

## ### 3. 與相鄰概念的關聯

#### #### 3.1 OSI 模型與 TCP/IP 參考模型

**定義/核心觀念：**
區域網路連線實作涉及網路通訊的各個層次。OSI（開放式系統互連）模型和 TCP/IP 參考模型是理解網路通訊層次結構的兩種主要框架。

**關聯：**
*   **物理層 (L1)：** 網路線、網卡、交換器的埠等屬於物理層，負責傳輸原始位元流。RJ45 接頭、乙太網路線規格 (Cat5e/6) 都在此層定義。
*   **資料連結層 (L2)：** 網卡本身和交換器主要在此層運作。網卡處理 MAC 位址（媒體存取控制位址）和乙太網路幀的封裝與解封裝。交換器則根據 MAC 位址進行精準轉發。
*   **網路層 (L3)：** IP 位址、子網路遮罩和預設閘道都是網路層的概念。路由器主要在此層運作，負責根據 IP 位址進行路由和轉發資料包。在我們的實作中，手動設定 IP 位址就是在配置這一層的資訊。
*   **傳輸層 (L4) 和應用層 (L5-L7)：** 區域網路連線實作為上層應用提供了基礎。例如，當您在 LAN 中分享檔案或瀏覽網頁時，檔案傳輸協定 (FTP)、超文本傳輸協定 (HTTP) 等應用層協定會在 TCP/UDP（傳輸層）之上運行，而這些都建立在底層已配置好的 LAN 連線上。

-----

#### #### 3.2 DHCP 與靜態 IP

**定義/核心觀念：**
在區域網路中，IP 位址的分配方式可以分為兩種：動態主機設定協定 (DHCP) 自動分配和手動設定靜態 IP。

**關聯：**
*   **DHCP (Dynamic Host Configuration Protocol)：** 是一種網路協定，用於自動分配網路裝置所需的 IP 位址、子網路遮罩、預設閘道和 DNS 伺服器等配置資訊。它極大地簡化了網路管理，尤其是在大型網路中。當您將電腦設定為「自動取得 IP 位址」時，您的電腦就會向網路中的 DHCP 伺服器發出請求。
*   **靜態 IP (Static IP)：** 是指手動為網路裝置設定固定的 IP 位址。這對於需要長期保持相同 IP 位址的裝置（如伺服器、網路印表機）非常有用，因為它們的位址不會隨 DHCP 租約到期而改變。

在本次實作中，我們選擇手動設定靜態 IP，這有助於學習者更深入理解 IP 位址、子網路遮罩和閘道的概念及其配置方式。在實際環境中，家庭和小型辦公室通常會使用路由器內建的 DHCP 伺服器來自動分配 IP。

-----

#### #### 3.3 LAN 與網際網路 (WAN) 的橋樑：路由器

**定義/核心觀念：**
路由器在區域網路中扮演著至關重要的角色，它不僅連接了 LAN 內部的裝置，更重要的是，它作為 LAN 和外部網際網路 (WAN) 之間的閘道。

**關聯：**
*   **內部連線：** 許多家用路由器都整合了交換器功能，其多個 LAN 埠可以讓多台電腦像連接到獨立交換器一樣互相通訊。
*   **外部連線：** 路由器的 WAN 埠連接到網際網路服務供應商 (ISP) 的設備 (如數據機)，負責將您區域網路內的資料流量導向網際網路，並將來自網際網路的流量導回您的 LAN。
*   **網路位址轉換 (NAT)：** 路由器通常會執行 NAT 功能，允許多台 LAN 內的裝置共享一個公共 IP 位址來存取網際網路，同時保護內部網路的隱私。

簡單來說，LAN 實現了內部設備的互相通訊，而路由器則將這個內部世界連接到了更廣闊的網際網路。

## ### 4. 進階內容

#### #### 4.1 網路線材的選擇與標準

**定義/核心觀念：**
乙太網路線材有不同的類別 (Category, Cat)，它們決定了網路線的頻寬和傳輸速度上限。選擇合適的線材對於確保網路效能至關重要。

**細節：**
*   **Cat5e (Category 5e)：** 增強型五類線。支援高達 1 Gbps (千兆位元每秒) 的速度，頻寬為 100 MHz。在多數家庭和小型辦公室環境中仍廣泛使用。
*   **Cat6 (Category 6)：** 六類線。支援高達 1 Gbps 的速度，但在較短距離 (約 55 公尺) 內可支援 10 Gbps。頻寬為 250 MHz，提供更好的抗干擾能力。
*   **Cat6a (Category 6a)：** 增強型六類線。在長達 100 公尺的距離上完全支援 10 Gbps 的速度。頻寬為 500 MHz，是未來 10 GbE 網路的常見選擇。
*   **Cat7 / Cat8：** 更高階的線材，支援更高的速度 (Cat7 支援 10 Gbps, Cat8 支援 25/40 Gbps) 和頻寬 (Cat7 600 MHz, Cat8 2000 MHz)，主要用於資料中心和高需求的企業環境。

**選擇建議：**
*   對於一般家庭和辦公室，如果現有設備都是千兆網卡，Cat5e 或 Cat6 已經足夠。
*   如果考慮未來升級到 2.5G 或 10G 網路，建議佈線時直接採用 Cat6a 或更高規格，以避免重複施工。

## ### 5. 常見錯誤與澄清

#### #### 5.1 網路連線相關錯誤

**定義/核心觀念：**
在實作區域網路連線時，初學者常會遇到一些問題，以下是一些常見錯誤及其解決方案。

**常見錯誤與澄清：**

1.  **網路線沒接好或接頭損壞：**
    *   **錯誤現象：** 網卡指示燈不亮，或交換器對應埠燈不亮。
    *   **澄清：** 確保 RJ45 接頭插入網卡或交換器埠時發出「咔噠」聲，並且牢固。檢查網路線本身是否有明顯物理損壞。可以嘗試更換一條已知良好的網路線。
    *   **實作提示：** 自己製作網路線時，需要確保 T568B 或 T568A 接線標準正確，且所有線芯都接觸良好。對於直通線，兩端都採用 T568B 或 T568A 即可。

2.  **IP 位址衝突 (IP Conflict)：**
    *   **錯誤現象：** 兩台或多台電腦設定了完全相同的 IP 位址，系統可能會彈出警告。
    *   **澄清：** 區域網路內的每台裝置都必須有唯一的 IP 位址。
    *   **解決方案：** 檢查所有手動設定 IP 的電腦，確保它們的 IP 位址各不相同。如果使用 DHCP，通常路由器會自動避免衝突，但有時在 DHCP 租約更新或多個 DHCP 伺服器並存時也可能發生。

3.  **子網路遮罩設定錯誤：**
    *   **錯誤現象：** 兩台 IP 位址看起來在同一個網段的電腦，卻無法互相通訊 (ping 不通)。
    *   **澄清：** 子網路遮罩決定了哪些 IP 位址在同一個邏輯子網路中。如果子網路遮罩設定不一致，即使 IP 位址看似在同一網段，系統也會誤判它們在不同網段，從而導致通訊失敗。例如，一台 `192.168.1.10` 使用 `255.255.255.0`，另一台 `192.168.1.11` 卻誤用 `255.255.0.0`，就會出問題。
    *   **解決方案：** 確保所有在同一區域網路中的裝置都使用相同的子網路遮罩 (例如 `255.255.255.0`)。

4.  **預設閘道設定錯誤：**
    *   **錯誤現象：** 區域網路內部可以通訊，但無法連接到網際網路。
    *   **澄清：** 預設閘道是將資料流量導向 LAN 外部（通常是網際網路）的唯一出口。如果閘道設定錯誤或沒有設定，資料包就不知道該往哪裡發送。
    *   **解決方案：** 確保預設閘道設定為路由器的 LAN 介面 IP 位址。如果僅作 LAN 內部通訊，則可以忽略此設定。

5.  **防火牆阻擋通訊：**
    *   **錯誤現象：** IP 配置正確，且能 ping 通，但某些應用程式（如檔案分享、遠端桌面）無法使用。
    *   **澄清：** 作業系統內建或第三方防火牆可能會阻止特定連接埠或協定的流量。
    *   **解決方案：** 檢查 Windows 防火牆或其他安全軟體的設定，確認所需服務的連接埠已允許通過。在測試階段，可以暫時關閉防火牆來排除問題，但務必在測試完成後重新啟用。

6.  **Hub 與 Switch 的混淆：**
    *   **錯誤現象：** 網路速度緩慢、頻繁斷線、碰撞嚴重。
    *   **澄清：** Hub 是共享頻寬的半雙工設備，會導致所有連接埠共享頻寬，並在任何時間只允許一個裝置傳輸，造成大量碰撞。Switch 是全雙工設備，每個埠都有專屬頻寬，且能夠智慧地轉發流量，大大提高效率。
    *   **解決方案：** 確保使用交換器 (Switch) 而非集線器 (Hub) 來構建現代區域網路。

## ### 6. 小練習 (附詳解)

#### #### 6.1 練習一：兩台電腦透過交換器互連並測試通訊

**目標：** 學習者能夠實際連接兩台電腦到交換器，並手動設定 IP 位址，最終驗證兩台電腦是否能互相通訊。

**所需設備：**
*   兩台電腦 (PC-A, PC-B)，均有乙太網路卡。
*   一台乙太網路交換器。
*   兩條乙太網路線 (直通線)。

**步驟：**

1.  **物理連線：**
    *   將交換器插上電源。
    *   使用一條網路線將 PC-A 的網卡連接到交換器上任意一個可用的埠 (例如 Port 1)。
    *   使用另一條網路線將 PC-B 的網卡連接到交換器上任意另一個可用的埠 (例如 Port 2)。
    *   確認交換器上 Port 1 和 Port 2 的指示燈都亮起，表示物理連線正常。

2.  **設定 PC-A 的 IP 位址：**
    *   在 PC-A 上，進入「網路和網際網路設定」->「乙太網路」->「變更介面卡選項」。
    *   右鍵點擊您的乙太網路介面卡，選擇「內容」。
    *   選中「網際網路通訊協定第 4 版 (TCP/IPv4)」，點擊「內容」。
    *   選擇「使用下列 IP 位址」，並輸入：
        *   **IP 位址：** `192.168.1.10`
        *   **子網路遮罩：** `255.255.255.0`
        *   **預設閘道：** `192.168.1.1` (此處作為佔位符，因為沒有路由器連外網，但保持格式完整)
        *   **慣用 DNS 伺服器：** 留空
    *   點擊「確定」保存設定。

3.  **設定 PC-B 的 IP 位址：**
    *   在 PC-B 上，重複步驟 2，但輸入不同的 IP 位址：
        *   **IP 位址：** `192.168.1.11`
        *   **子網路遮罩：** `255.255.255.0`
        *   **預設閘道：** `192.168.1.1` (與 PC-A 相同)
        *   **慣用 DNS 伺服器：** 留空
    *   點擊「確定」保存設定。

4.  **測試連線 (從 PC-A)：**
    *   在 PC-A 上，按下 `Win + R`，輸入 `cmd`，然後按 Enter 打開命令提示字元。
    *   輸入指令 `ping 192.168.1.11`，然後按 Enter。
    *   觀察 ping 的結果。

5.  **測試連線 (從 PC-B)：**
    *   在 PC-B 上，重複步驟 4。
    *   輸入指令 `ping 192.168.1.10`，然後按 Enter。
    *   觀察 ping 的結果。

**詳解：**

*   **Ping 結果判讀：**
    *   如果看到類似 `回覆自 192.168.1.11: 位元組=32 時間<1ms TTL=128` 這樣的輸出，並且有 4 個封包傳送、4 個接收，則表示 PC-A 成功地與 PC-B 建立通訊。
    *   如果看到 `要求等候逾時` (Request timed out) 或 `目的主機無法連線` (Destination host unreachable)，則表示連線失敗。需要檢查以下幾點：
        *   **物理連線：** 網路線是否插牢？交換器指示燈是否亮起？
        *   **IP 設定：** 兩台電腦的 IP 位址是否在同一網段 (前三段相同) 且各不相同？子網路遮罩是否一致 (通常是 `255.255.255.0`)？
        *   **防火牆：** 兩台電腦的 Windows 防火牆或其他安全軟體是否阻擋了 ICMP (ping 使用的協定) 流量？可以暫時關閉防火牆進行測試。
*   **預期結果：** 兩台電腦都應該能夠成功 ping 到對方，這證明區域網路連線已成功建立並可正常通訊。

#### #### 6.2 練習二：使用 `ipconfig` 命令檢查網路配置

**目標：** 學習者能夠使用命令列工具快速查看電腦的網路介面卡配置資訊。

**所需設備：**
*   一台已配置好網路的電腦 (可沿用練習一中的 PC-A 或 PC-B)。

**步驟：**

1.  **打開命令提示字元：**
    *   按下 `Win + R`，輸入 `cmd`，然後按 Enter。

2.  **執行 `ipconfig` 命令：**
    *   在命令提示字元中，輸入 `ipconfig`，然後按 Enter。

3.  **執行 `ipconfig /all` 命令：**
    *   在命令提示字元中，輸入 `ipconfig /all`，然後按 Enter。

**詳解：**

*   **`ipconfig` 命令輸出：**
    *   此命令會顯示每個網路介面卡的基本 IP 配置資訊，包括：
        *   **IPv4 位址：** 您設定的 IP 位址 (例如 `192.168.1.10`)。
        *   **子網路遮罩：** 您設定的子網路遮罩 (例如 `255.255.255.0`)。
        *   **預設閘道：** 您設定的預設閘道 (例如 `192.168.1.1`)。
    *   這是一個快速確認您的 IP 設定是否正確的工具。

*   **`ipconfig /all` 命令輸出：**
    *   此命令會顯示更詳細的配置資訊，包括：
        *   **主機名稱 (Host Name)：** 電腦的名稱。
        *   **實體位址 (Physical Address / MAC 位址)：** 網卡的全球唯一硬體位址。
        *   **是否啟用 DHCP (DHCP Enabled)：** 顯示網卡是否設定為自動取得 IP。
        *   **DHCP 伺服器：** 如果使用 DHCP，會顯示 DHCP 伺服器的 IP 位址。
        *   **DNS 伺服器：** 設定的 DNS 伺服器 IP 位址。
        *   還有其他如網卡描述、連線狀態等資訊。
    *   使用 `ipconfig /all` 可以獲得更全面的網路配置概覽，對於問題排查非常有用。

**預期結果：**
您會看到與您在練習一中手動設定的 IP 位址、子網路遮罩和預設閘道相符的資訊，以及您的網卡 MAC 位址等其他詳細資料。

## ### 7. 延伸閱讀與參考

**建議書籍/資源：**

*   **《電腦網路：概念與實作 (Computer Networking: A Top-Down Approach)》**：經典的網路基礎教材，深入淺出地介紹網路原理。
*   **《TCP/IP 詳盡指南》系列 (TCP/IP Illustrated Series)**：如果您想深入了解 TCP/IP 協定的細節，這是權威的參考書。
*   **CCNA 認證教材**：思科 (Cisco) 的網路認證教材，包含了豐富的網路基礎知識和實作案例，對於理解區域網路和路由器的配置非常有幫助。
*   **網路教學網站：**
    *   **鳥哥的 Linux 私房菜：** 其中有關於 Linux 網路設定的章節，對於理解網路原理和實際操作有益。
    *   **Microsoft Learn：** 提供 Windows 網路配置的官方文件和教學。

**網路工具：**

*   **Wireshark：** 一款強大的網路封包分析工具，可以擷取和分析網路流量，幫助您深入理解資料在網路中的傳輸過程。
*   **`netstat`：** 命令列工具，用於顯示網路連線、路由表和網路介面統計資訊。
*   **`tracert` (Windows) / `traceroute` (Linux/macOS)：** 命令列工具，用於追蹤資料包從來源到目的地的路徑，顯示途中經過的每一個路由器跳點。

**RFC (Request for Comments) 文件：**
RFC 是定義網際網路標準的技術文件。對於更深層次的理解，您可以查閱以下相關 RFC：
*   RFC 791: Internet Protocol (IP)
*   RFC 792: Internet Control Message Protocol (ICMP)
*   RFC 826: An Ethernet Address Resolution Protocol (ARP)
*   RFC 2131: Dynamic Host Configuration Protocol (DHCP)
*   RFC 8941: IPv4 Subnetting and Supernetting (雖然這是較老的標準，但概念仍然適用於理解子網路遮罩)

這些資源將幫助您從理論到實作，全面提升對區域網路的理解。
