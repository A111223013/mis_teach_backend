# 電腦科學概論

### 1. 什麼是電腦科學？

#### 1.1 定義與核心觀念

電腦科學（Computer Science, CS）是一門研究計算機及其應用、以及計算理論、設計、實現與應用的學科。它不僅僅是學習如何使用電腦或寫程式，更深入探討的是資訊的本質、計算的極限以及如何有效地解決問題。

*   **核心定義**：電腦科學是一門研究計算機系統和計算過程的科學，包括其理論、設計、開發和應用。
*   **核心觀念**：
    *   **演算法（Algorithms）**：解決問題的明確步驟序列，是電腦科學的靈魂。
    *   **資料結構（Data Structures）**：組織和儲存資料的方式，影響演算法的效率。
    *   **計算理論（Theory of Computation）**：研究計算模型及其能力和限制。
    *   **程式語言（Programming Languages）**：設計用於與電腦溝通的正式語言。
    *   **電腦架構（Computer Architecture）**：電腦系統的設計與組織方式。
    *   **作業系統（Operating Systems）**：管理電腦硬體與軟體資源的系統軟體。
    *   **網路（Networking）**：電腦之間資料交換的系統。
    *   **人工智慧（Artificial Intelligence, AI）**：使機器具備類人智慧的能力。

*   **與相鄰概念的關聯**：
    *   **與數學**：電腦科學的許多理論基礎源於數學（如邏輯、離散數學、機率統計）。演算法的分析也高度依賴數學工具。
    *   **與工程學**：電腦工程（Computer Engineering）更側重於硬體的設計與製造。軟體工程（Software Engineering）則專注於大規模軟體系統的開發與維護。電腦科學提供這些工程領域的理論基礎。
    *   **與哲學/認知科學**：探討心智與計算的關係，以及人工智慧的倫理和社會影響。

#### 1.2 電腦科學的歷史脈絡

電腦科學的發展是一段從抽象思維到實體機器的演化過程。

*   **早期計算工具**：
    *   古老的算盤、算籌等是人類早期對計算效率提升的嘗試。
    *   17世紀，**布萊茲·帕斯卡（Blaise Pascal）**發明了帕斯卡計算器，用於加減法。
    *   17世紀末，**戈特弗里德·萊布尼茨（Gottfried Leibniz）**設計了可進行乘除法的階梯計算器。
*   **計算理論的萌芽**：
    *   19世紀，**查爾斯·巴貝奇（Charles Babbage）**設計了分析機（Analytical Engine），被視為第一台通用電腦的設計藍圖，引入了程式（穿孔卡片）和記憶體概念。
    *   **埃達·洛芙萊斯（Ada Lovelace）**為分析機編寫了第一個演算法，被譽為世界上第一位程式設計師。
    *   20世紀初，**阿蘭·圖靈（Alan Turing）**提出了**圖靈機（Turing Machine）**的抽象模型，確立了「可計算性」的理論基礎，是現代電腦的數學模型。
    *   **約翰·馮·紐曼（John von Neumann）**提出了**馮紐曼架構（Von Neumann Architecture）**，統一了指令和數據的儲存方式，奠定了現代電腦的硬體基礎。
*   **電子計算機的誕生與發展**：
    *   20世紀40年代，**ENIAC**（電子數字積分器與計算器）標誌著第一代電子計算機的出現，體積龐大，使用真空管。
    *   隨後，電晶體、積體電路、微處理器的發明，使電腦體積縮小、性能提升、成本降低。
    *   個人電腦、網際網路的普及，將電腦帶入千家萬戶，催生了資訊時代。
*   **現代與未來**：
    *   雲端運算、大數據、人工智慧、物聯網等新技術的興起，持續推動電腦科學的邊界。

### 2. 電腦如何運作？ - 基本原理

電腦，無論大小，其核心運作原理都基於對數位資訊的處理。

#### 2.1 資料的表示：位元與位元組

電腦是數位機器，它只認識兩種狀態：開（電壓高）和關（電壓低），通常表示為1和0。

*   **定義**：
    *   **位元（Bit）**：是資訊的最小單位，只能表示0或1。Bit是二進制數字（Binary Digit）的縮寫。
    *   **位元組（Byte）**：是電腦儲存和處理資料的基本單位，通常由8個位元組成。
*   **核心觀念**：
    *   **二進制系統（Binary System）**：電腦內部所有資訊都以二進制（Base-2）形式表示。例如，十進制數字 $5$ 在二進制中是 $101_2$。
    *   **數位化（Digitalization）**：將現實世界的類比資訊（如聲音、圖像、文字）轉換成數位形式（二進制資料）的過程。
*   **典型例子與轉換**：
    *   **數字表示**：
        *   正整數：直接轉換二進制。例如，十進制 $13$ 轉換為二進制是 $1101_2$。
        *   負數：常用二補數（Two's Complement）表示，方便電腦進行加減運算。
        *   浮點數：依照IEEE 754標準，用科學記號法表示（例如 $3.14 \times 10^0$）。
    *   **文字表示**：
        *   **ASCII（American Standard Code for Information Interchange）**：最早的標準，用7或8個位元表示英文字母、數字和符號。例如，大寫字母 'A' 的ASCII碼是65，二進制為 $01000001_2$。
        *   **Unicode**：為了解決ASCII無法表示多國語言的問題而生。它為世界上幾乎所有文字系統中的每個字元都分配了一個唯一的數字，例如 '中' 字的Unicode碼是 `U+4E2D`。UTF-8是Unicode最常見的編碼方式。
    *   **圖片與聲音**：
        *   **圖片**：由像素（Pixel）組成。每個像素的顏色深度（Bit Depth）決定了它能表示多少種顏色。例如，24位元真彩色每個像素用24位元表示，可顯示 $2^{24}$ 種顏色。
        *   **聲音**：透過取樣（Sampling）將聲波在時間上離散化，量化（Quantization）將每個取樣點的振幅轉換為數位值。取樣頻率和位元深度決定了聲音的品質。
*   **與相鄰概念的關聯**：
    *   **與電腦硬體**：硬體電路設計（如邏輯閘）直接操作位元。處理器、記憶體、儲存裝置都以位元和位元組為單位來處理資料。
    *   **與程式語言**：程式語言提供資料型態（如整數、浮點數、字元），這些型態在底層都對應著特定的二進制表示方式。

#### 2.2 電腦硬體架構：馮紐曼架構

現代電腦的核心架構，幾乎都遵循由約翰·馮·紐曼於1945年提出的「馮紐曼架構」。

*   **定義**：馮紐曼架構將程式指令和資料儲存在同一個記憶體空間中，並透過中央處理器（CPU）來執行這些指令。
*   **核心觀念**：
    *   **儲存程式概念（Stored-Program Concept）**：程式指令和資料一樣，都儲存在記憶體中，可以被CPU讀取和修改。這使得電腦具有通用性，不需重新接線就能執行不同任務。
    *   **核心組件**：
        *   **中央處理器（Central Processing Unit, CPU）**：電腦的大腦，負責執行指令、進行運算和控制其他部件。
            *   **算術邏輯單元（Arithmetic Logic Unit, ALU）**：執行算術運算（加減乘除）和邏輯運算（AND, OR, NOT）。
            *   **控制單元（Control Unit, CU）**：解釋指令、協調電腦各部件工作。
            *   **暫存器（Registers）**：CPU內部極高速的少量儲存單元，用於暫存指令、數據和位址。
        *   **記憶體（Memory，通常指RAM, Random Access Memory）**：暫時儲存程式和資料的地方，速度快但斷電後資料消失。
        *   **輸入裝置（Input Devices）**：將外部資訊轉換為電腦可識別的數位訊號（如鍵盤、滑鼠、麥克風）。
        *   **輸出裝置（Output Devices）**：將電腦處理後的資訊轉換為人類可理解的形式（如螢幕、印表機、喇叭）。
        *   **儲存裝置（Storage Devices）**：長期儲存資料，斷電後資料依然保留（如硬碟HHD, 固態硬碟SSD, 隨身碟）。
    *   **匯流排（Bus）**：連接各個部件的電子通路，負責傳輸資料、位址和控制訊號。
*   **運作流程（指令週期 - Fetch-Decode-Execute Cycle）**：
    1.  **取指令（Fetch）**：CPU從記憶體中取出下一條指令。
    2.  **解碼指令（Decode）**：控制單元解釋指令的含義。
    3.  **執行指令（Execute）**：ALU或其他單元執行指令所指定的運算。
    4.  **寫回結果（Write-back）**：將運算結果存回記憶體或暫存器。
*   **與相鄰概念的關聯**：
    *   **與作業系統**：作業系統是管理硬體資源和執行應用程式的軟體。它透過CPU提供的指令集來控制硬體。
    *   **與程式語言**：高階程式語言編寫的程式最終會被編譯或直譯成CPU可以直接執行的機器碼（Machine Code）。

#### 2.3 軟體：作業系統與應用程式

電腦的價值不僅在於其硬體，更在於能透過軟體執行各種任務。

*   **定義**：
    *   **軟體（Software）**：一系列指令的集合，指揮硬體完成特定任務。軟體可分為系統軟體和應用軟體。
    *   **作業系統（Operating System, OS）**：是系統軟體的核心，負責管理電腦硬體資源、提供使用者介面，並為應用程式提供運行環境和服務。
    *   **應用程式（Application Software）**：為了解決特定使用者需求而設計的軟體，運行在作業系統之上。
*   **核心觀念**：
    *   **系統資源管理**：
        *   **行程管理（Process Management）**：作業系統分配CPU時間給不同的程式（行程），實現多工處理。
        *   **記憶體管理（Memory Management）**：分配記憶體給各個程式，防止衝突，並在需要時使用虛擬記憶體技術。
        *   **檔案系統（File System）**：管理資料在儲存裝置上的組織、存取和儲存方式。
        *   **裝置管理（Device Management）**：管理輸入/輸出裝置，讓應用程式能透過統一介面使用硬體。
    *   **使用者介面（User Interface, UI）**：作業系統提供與使用者互動的方式，可以是圖形使用者介面（GUI）或命令列介面（CLI）。
    *   **應用程式介面（Application Programming Interface, API）**：作業系統提供給應用程式開發者的介面，讓應用程式可以呼叫作業系統的功能。
*   **典型例子**：
    *   **作業系統**：Windows, macOS, Linux, Android, iOS。
    *   **應用程式**：Microsoft Word (文書處理), Google Chrome (網頁瀏覽器), Photoshop (影像處理), 各種遊戲、手機App等。
*   **與相鄰概念的關聯**：
    *   **與硬體**：作業系統是硬體與應用程式之間的橋樑，沒有作業系統，硬體就只是一堆無用的電子元件，應用程式也無法運行。
    *   **與程式語言**：程式設計師使用程式語言編寫應用程式，這些應用程式最終需要作業系統來執行。

### 3. 計算的靈魂：演算法與程式設計

演算法是電腦科學的核心，而程式設計則是將演算法具體實現的藝術。

#### 3.1 演算法是什麼？

*   **定義**：演算法（Algorithm）是一組明確定義、有限步驟的指令序列，用於解決特定問題或執行特定任務。
*   **核心觀念**：
    *   **輸入（Input）**：演算法接受零個或多個輸入。
    *   **輸出（Output）**：演算法產生一個或多個輸出。
    *   **明確性（Definiteness）**：每個步驟都必須清晰且無歧義。
    *   **有限性（Finiteness）**：演算法必須在有限的步驟後終止。
    *   **有效性（Effectiveness）**：每個步驟都必須是基本且可執行的。
    *   **正確性（Correctness）**：演算法必須產生正確的結果。
    *   **效率（Efficiency）**：衡量演算法使用資源（時間、空間）的多少。
*   **典型例子**：
    *   **食譜**：做菜的步驟序列，接收食材（輸入）並產生一道菜（輸出）。
    *   **尋路**：從地圖上的一個點到另一個點的最短路徑。
    *   **排序演算法**：給定一個數字列表，將其按照大小順序排列（如冒泡排序、快速排序）。
    *   **搜尋演算法**：在一個列表中找到特定元素的位置（如線性搜尋、二分搜尋）。
*   **推導**：
    1.  **問題分析**：明確問題是什麼？有哪些輸入？期望的輸出是什麼？
    2.  **設計演算法**：思考解決問題的步驟。可以透過流程圖、偽程式碼（Pseudocode）來描述。
    3.  **演算法分析**：評估演算法的效率，通常用大O符號（Big O notation）表示時間複雜度和空間複雜度。例如，線性搜尋的時間複雜度是 $O(n)$，二分搜尋是 $O(\log n)$。
    4.  **演算法實現**：使用程式語言將演算法轉換為可執行的程式碼。

#### 3.2 程式語言與程式設計

*   **定義**：程式語言（Programming Language）是一種用於向電腦發出指令以執行特定任務的正式語言。程式設計（Programming）是使用程式語言編寫、測試、除錯和維護程式碼的過程。
*   **核心觀念**：
    *   **高階語言（High-Level Languages）**：更接近人類語言和數學表達，易於理解和編寫，例如Python, Java, C++, JavaScript。
    *   **低階語言（Low-Level Languages）**：直接操作電腦硬體，效率高但難以編寫。包括機器語言（Machine Language，CPU直接執行的二進制碼）和組合語言（Assembly Language，機器語言的符號表示）。
    *   **編譯器（Compiler）**：將高階程式碼一次性轉換為機器碼的軟體。
    *   **直譯器（Interpreter）**：逐行讀取並執行高階程式碼的軟體。
    *   **語法（Syntax）**：程式語言的規則，規定程式碼如何寫才合法。
    *   **語義（Semantics）**：程式碼的含義，程式碼執行後會發生什麼。
*   **典型例子**：
    *   **Python**：語法簡潔，常用於資料科學、人工智慧、網路開發、自動化。
    *   **Java**：跨平台（Write Once, Run Anywhere），常用於企業級應用、Android開發。
    *   **C++**：性能高，常用於遊戲開發、作業系統、嵌入式系統。
    *   **JavaScript**：主要用於網頁前端開發，也可以用於後端（Node.js）。
*   **推導：從問題到演算法，從演算法到程式碼**：
    1.  **問題**：計算從1加到100的總和。
    2.  **演算法（偽程式碼）**：
        ```
        Function sum_1_to_N(N):
            Initialize total_sum = 0
            For each number from 1 to N:
                Add number to total_sum
            Return total_sum
        ```
    3.  **程式碼（Python）**：
        ```python
        def sum_1_to_N(n):
            total_sum = 0
            for number in range(1, n + 1):
                total_sum += number
            return total_sum

        print(sum_1_to_N(100)) # 輸出 5050
        ```
*   **與相鄰概念的關聯**：
    *   **與演算法**：程式語言是實現演算法的工具。一個好的演算法可以用任何程式語言實現，但程式語言的特性會影響實現的便利性和效率。
    *   **與資料結構**：程式設計師使用資料結構來組織和管理程式中的資料。選擇合適的資料結構對於演算法的效率至關重要。

#### 3.3 資料結構

*   **定義**：資料結構（Data Structure）是組織和儲存資料的方式，以便能夠高效地存取和修改。
*   **核心觀念**：
    *   **邏輯結構**：資料之間的抽象關係（如線性、樹狀、圖狀）。
    *   **物理結構**：資料在記憶體中的實際儲存方式。
    *   **操作**：對資料結構進行的常見操作（如插入、刪除、搜尋、更新）。
*   **典型例子**：
    *   **陣列（Array）**：一系列相同型態的元素在記憶體中連續儲存，透過索引直接存取。
    *   **連結串列（Linked List）**：元素在記憶體中不一定連續，每個元素（節點）包含資料和指向下一個節點的指標。
    *   **堆疊（Stack）**：後進先出（LIFO）的資料結構，像一疊盤子。
    *   **佇列（Queue）**：先進先出（FIFO）的資料結構，像排隊。
    *   **樹（Tree）**：階層式結構，由節點和邊組成，根節點、父節點、子節點等。
        *   **二元搜尋樹（Binary Search Tree, BST）**：一種特殊的樹，左子樹所有節點的值小於根節點，右子樹所有節點的值大於根節點，利於快速搜尋。
    *   **圖（Graph）**：由頂點（Vertex）和邊（Edge）組成的非線性結構，用於表示複雜的關係（如社交網路）。
    *   **雜湊表（Hash Table）**：透過雜湊函數將鍵（Key）映射到儲存位置，實現快速存取。
*   **與演算法的關聯**：
    *   資料結構與演算法是相輔相成的。選擇正確的資料結構可以大大簡化演算法的設計，並提高其執行效率。
    *   例如，在一個未排序的列表中搜尋元素需要線性搜尋 $O(n)$，但如果資料儲存在二元搜尋樹中，搜尋效率可達 $O(\log n)$。

### 4. 網路與資訊安全簡介

現代電腦幾乎都離不開網路，而資訊安全則是保障網路世界運作的基石。

#### 4.1 電腦網路基礎

*   **定義**：電腦網路（Computer Network）是指兩台或更多台電腦或其他電子設備透過傳輸介質（如電纜、無線電波）連接起來，以分享資源和交換資訊的系統。
*   **核心觀念**：
    *   **IP位址（Internet Protocol Address）**：每台連接到網路的設備的唯一識別碼，用於在網路上定位設備。例如 `192.168.1.1` (IPv4) 或 `2001:0db8:85a3:0000:0000:8a2e:0370:7334` (IPv6)。
    *   **通訊協定（Protocol）**：一套規則和標準，規範網路設備之間如何溝通和交換資料。
        *   **TCP/IP（Transmission Control Protocol/Internet Protocol）**：網際網路的核心協定套件。
            *   **IP** 負責封包的路由和尋址。
            *   **TCP** 提供可靠的資料傳輸，確保資料包的正確順序和完整性。
        *   **HTTP（Hypertext Transfer Protocol）**：用於萬維網上傳輸超文本，即網頁內容。
        *   **DNS（Domain Name System）**：將易於記憶的網域名稱（如 `www.google.com`）轉換為IP位址。
    *   **客戶端/伺服器模型（Client-Server Model）**：一種分散式應用程式結構，其中客戶端（如網頁瀏覽器）向伺服器（如網站伺服器）請求服務或資源。
    *   **封包交換（Packet Switching）**：資料被分割成小塊（封包），獨立傳輸，再在目的地組裝。這是網際網路的基礎。
    *   **網路類型**：
        *   **區域網路（Local Area Network, LAN）**：覆蓋小範圍地理區域（如家庭、辦公室）。
        *   **廣域網路（Wide Area Network, WAN）**：覆蓋大範圍地理區域，如網際網路。
*   **典型例子**：
    *   **網際網路（Internet）**：全球性的電腦網路，連接了數十億設備。
    *   **Wi-Fi**：一種無線區域網路技術，允許設備無線連接到網路。
    *   **電子郵件**：透過網路傳輸訊息的應用。
    *   **網頁瀏覽**：透過HTTP協定存取網站。
*   **與相鄰概念的關聯**：
    *   **與作業系統**：作業系統包含網路堆疊（Network Stack），處理網路通訊的底層細節。
    *   **與資料表示**：網路傳輸的資料最終還是以位元的形式傳輸。

#### 4.2 資訊安全簡介

*   **定義**：資訊安全（Information Security）是指保護資訊系統及其所儲存、處理和傳輸的資訊，使其免受未經授權的存取、使用、揭露、破壞、修改或毀壞。
*   **核心觀念**：資訊安全的目標通常透過以下三個原則來概括（CIA三要素）：
    *   **機密性（Confidentiality）**：確保只有經過授權的人才能存取資訊。
    *   **完整性（Integrity）**：確保資訊在儲存和傳輸過程中未被未經授權地修改，保持其準確性和完整性。
    *   **可用性（Availability）**：確保授權使用者在需要時可以存取資訊和系統。
*   **常見威脅**：
    *   **惡意軟體（Malware）**：
        *   **病毒（Virus）**：感染其他程式並自我複製。
        *   **蠕蟲（Worm）**：透過網路自我複製和傳播。
        *   **木馬程式（Trojan Horse）**：偽裝成有用軟體，實則執行惡意操作。
        *   **勒索軟體（Ransomware）**：加密使用者資料並勒索贖金。
    *   **網路釣魚（Phishing）**：冒充合法實體誘騙使用者提供敏感資訊（如帳號密碼）。
    *   **阻斷服務攻擊（Denial of Service, DoS / Distributed Denial of Service, DDoS）**：透過大量流量或請求淹沒目標系統，使其無法提供正常服務。
    *   **中間人攻擊（Man-in-the-Middle Attack, MitM）**：攻擊者截獲並可能修改通訊雙方之間的訊息。
    *   **弱密碼與帳戶盜用**：因密碼過於簡單或重複使用導致帳戶被盜。
*   **基本防禦措施**：
    *   **防火牆（Firewall）**：監控和控制網路流量，阻止未經授權的存取。
    *   **防毒軟體（Antivirus Software）**：檢測和清除惡意軟體。
    *   **加密（Encryption）**：將資料轉換為密文，只有擁有金鑰的人才能解密，確保機密性。
    *   **身分驗證（Authentication）**：確認使用者身分（如密碼、多因素驗證）。
    *   **備份與復原**：定期備份重要資料，以應對資料丟失或損壞。
*   **與相鄰概念的關聯**：
    *   **與網路**：網路是資訊安全的主要攻擊面，大部分安全威脅都發生在網路層面。
    *   **與演算法**：密碼學（Cryptography）是資訊安全的重要組成部分，它依賴於數學演算法來實現加密、解密和雜湊等功能。

### 5. 小練習（附詳解）

#### 練習一：二進制轉換與文字編碼

**問題**：
1.  將十進制數字 $27$ 轉換為8位元二進制表示。
2.  將二進制數字 $10110_2$ 轉換為十進制。
3.  假設使用ASCII編碼，字母 'C' 的十進制ASCII碼是67，請寫出其8位元二進制表示。

**詳解**：

1.  **將十進制數字 $27$ 轉換為8位元二進制表示。**
    *   **步驟一：連續除以2取餘數。**
        *   $27 \div 2 = 13$ 餘 $1$
        *   $13 \div 2 = 6$ 餘 $1$
        *   $6 \div 2 = 3$ 餘 $0$
        *   $3 \div 2 = 1$ 餘 $1$
        *   $1 \div 2 = 0$ 餘 $1$
    *   **步驟二：將餘數從下往上排列。**
        *   得到 $11011_2$。
    *   **步驟三：補足8位元。**
        *   在前面補 $0$ 直到8位元：$00011011_2$。

2.  **將二進制數字 $10110_2$ 轉換為十進制。**
    *   **步驟一：從右往左，將每個位元乘以 $2$ 的相應次方。**
        *   $0 \times 2^0 = 0 \times 1 = 0$
        *   $1 \times 2^1 = 1 \times 2 = 2$
        *   $1 \times 2^2 = 1 \times 4 = 4$
        *   $0 \times 2^3 = 0 \times 8 = 0$
        *   $1 \times 2^4 = 1 \times 16 = 16$
    *   **步驟二：將所有結果相加。**
        *   $0 + 2 + 4 + 0 + 16 = 22$。
    *   所以，$10110_2 = 22_{10}$。

3.  **假設使用ASCII編碼，字母 'C' 的十進制ASCII碼是67，請寫出其8位元二進制表示。**
    *   **步驟一：將十進制 $67$ 轉換為二進制（與問題1類似）。**
        *   $67 \div 2 = 33$ 餘 $1$
        *   $33 \div 2 = 16$ 餘 $1$
        *   $16 \div 2 = 8$ 餘 $0$
        *   $8 \div 2 = 4$ 餘 $0$
        *   $4 \div 2 = 2$ 餘 $0$
        *   $2 \div 2 = 1$ 餘 $0$
        *   $1 \div 2 = 0$ 餘 $1$
    *   **步驟二：將餘數從下往上排列。**
        *   得到 $1000011_2$。
    *   **步驟三：補足8位元。**
        *   在前面補 $0$ 直到8位元：$01000011_2$。

#### 練習二：演算法設計 - 尋找陣列中的最大值

**問題**：
設計一個演算法，在一個包含 $N$ 個整數的陣列中，找到並輸出最大的那個數。請以偽程式碼描述演算法步驟。

**詳解**：

**演算法名稱**：`FindMaximumInArray`

**輸入**：
*   一個整數陣列 `arr`
*   陣列的長度 `N`

**輸出**：
*   陣列中的最大值 `max_value`

**演算法步驟（偽程式碼）**：

1.  **檢查陣列是否為空**：
    *   如果 `N` 小於或等於 `0`，則陣列為空。此時無法找到最大值，可以輸出錯誤訊息或返回一個特殊值（例如負無限大），然後終止演算法。
2.  **初始化最大值**：
    *   將陣列的第一個元素 `arr[0]` 設為當前的最大值 `max_value`。
3.  **迭代遍歷陣列**：
    *   從陣列的第二個元素（索引 `1`）開始，到陣列的最後一個元素（索引 `N-1`）為止，執行以下操作：
        *   比較當前元素 `arr[i]` 與 `max_value`。
        *   如果 `arr[i]` 大於 `max_value`，則將 `max_value` 更新為 `arr[i]`。
4.  **返回最大值**：
    *   當遍歷完所有元素後，`max_value` 就是陣列中的最大值，將其返回。

**範例**：
假設輸入陣列 `arr = [10, 5, 20, 8, 15]`，`N = 5`。

1.  陣列不為空。
2.  `max_value` 初始化為 `arr[0]`，即 `10`。
3.  開始遍歷：
    *   `i = 1`：`arr[1]` 是 `5`。`5` 不大於 `10`。`max_value` 仍為 `10`。
    *   `i = 2`：`arr[2]` 是 `20`。`20` 大於 `10`。`max_value` 更新為 `20`。
    *   `i = 3`：`arr[3]` 是 `8`。`8` 不大於 `20`。`max_value` 仍為 `20`。
    *   `i = 4`：`arr[4]` 是 `15`。`15` 不大於 `20`。`max_value` 仍為 `20`。
4.  遍歷結束，返回 `max_value`，即 `20`。

### 6. 常見錯誤與澄清

*   **電腦科學只是寫程式嗎？**
    *   **澄清**：這是最常見的誤解。程式設計是電腦科學的一個重要組成部分，它是實現演算法和構建軟體的手段。然而，電腦科學更廣泛，它包含理論（如計算理論、演算法分析）、數學基礎、硬體設計、系統架構、人工智慧、資料庫、網路、資訊安全等多個領域。許多電腦科學家可能不寫程式，而是專注於理論研究或高層次系統設計。
*   **演算法一定要用程式碼實現嗎？**
    *   **澄清**：不是。演算法是一種抽象的解決問題的步驟序列，它可以透過自然語言、流程圖、偽程式碼等多種方式來描述。程式碼只是演算法的一種具體實現。在設計演算法的階段，重點在於邏輯和效率，而不是特定的程式語言語法。
*   **位元組和位元組的混淆。**
    *   **澄清**：在中文語境中，"位元"（bit）和 "位元組"（byte）有時容易混淆。請記住：
        *   **位元（bit）** 是最小單位，只有 $0$ 或 $1$。通常用小寫 'b' 表示，如 `10 Mbps` (每秒10兆位元)。
        *   **位元組（byte）** 通常是 $8$ 個位元。通常用大寫 'B' 表示，如 `10 MB` (10兆位元組)。
        *   $1$ Byte = $8$ bits。這兩者之間的區別非常重要，尤其是在測量網路速度（通常用bps）和儲存空間（通常用B）時。

    *   **《深入理解計算機系統》(Computer Systems: A Programmer's Perspective)** by Randal E. Bryant and David R. O'Hallaron：從程式設計師的角度深入介紹電腦系統的運作方式。
